Git in Termux: A Clean Workflow for Projects

Use Git confidently from your phone: simple habits that prevent losing work and keep your repos professional.

Updated: 2026-01-11 • Hands-on Git guide

Why you can trust this guide

It uses standard Git conventions (small commits, branches, readable messages) and avoids risky shortcuts.

1) Install Git and set your identity

Set name/email once so your commits look professional.

pkg install -y git

git config --global user.name "Your Name"
git config --global user.email "you@example.com"

2) Clone + keep a clean Projects folder

Put all repos in ~/Projects so backups are easy and paths are predictable.

mkdir -p ~/Projects && cd ~/Projects
git clone https://github.com/USER/REPO.git
cd REPO

3) The daily loop: pull → change → commit

Make small changes, commit often, and write messages that explain the why (not only the what).

git pull
# edit files

git status
git add .
git commit -m "Fix blog post structure and headings"
# then push when ready
# git push

4) Use branches for experiments (so main stays stable)

Branches let you try ideas safely. If it works, merge. If it fails, delete the branch.

git checkout -b feature/new-posts
# work...

git checkout main
git merge feature/new-posts

5) GitHub auth: prefer SSH keys

HTTPS can work with tokens, but SSH keys are smoother once set up (especially on mobile).

ssh-keygen -t ed25519 -C "github-termux"
cat ~/.ssh/id_ed25519.pub
# Add the public key to GitHub, then:
git remote set-url origin git@github.com:USER/REPO.git

6) .gitignore: keep junk out of your repo

Never commit secrets, build outputs, caches, or large temporary files.

nano .gitignore

# Python
__pycache__/
*.pyc
.venv/

# Termux / local
.DS_Store
*.log
secrets.env
