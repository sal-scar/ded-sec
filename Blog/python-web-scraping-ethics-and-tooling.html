1) The golden rules (so you don’t get blocked)

Scrape only public pages, respect robots.txt where applicable, and rate-limit your requests. Most bans happen because scripts hammer servers.

  • Add delays (e.g., 1–3 seconds)
  • Identify your scraper via User-Agent
  • Prefer official APIs when available

2) Setup in Termux (requests + bs4)

Start with requests + BeautifulSoup. Add lxml if you need faster parsing.

pkg install -y python
python -m pip install -U pip
python -m pip install requests beautifulsoup4 lxml

3) A basic scraper with retries + timeout

Always set timeouts and handle errors. Real websites fail sometimes.

import time
import requests
from bs4 import BeautifulSoup

URL = "https://example.com"

for attempt in range(3):
    try:
        r = requests.get(
            URL,
            timeout=15,
            headers={"User-Agent": "DedSecScraper/1.0 (+https://ded-sec.space/)"},
        )
        r.raise_for_status()
        soup = BeautifulSoup(r.text, "lxml")
        title = soup.title.get_text(strip=True) if soup.title else "(no title)"
        print("OK:", title)
        break
    except Exception as e:
        print("retry", attempt + 1, "->", e)
        time.sleep(2)
else:
    print("failed after 3 attempts")

4) Pagination without pain

Scrapers often fail because pagination is inconsistent. Prefer ‘next page’ links or page parameters you can verify in the URL.

Tip

If the site has JSON-LD (script type="application/ld+json"), parse that first. It’s usually cleaner than HTML.

5) Export data (CSV)

Export early. A CSV file makes debugging easier than printing to the terminal forever.

import csv
rows = [{"name":"Item A","price":10},{"name":"Item B","price":12}]
with open("out.csv","w",newline="",encoding="utf-8") as f:
    w = csv.DictWriter(f, fieldnames=rows[0].keys())
    w.writeheader(); w.writerows(rows)
print("saved out.csv")

Ethical scraping basics (in plain English)

Scraping is just automated reading — but the impact can be very different from a human browsing a page. Good scraping is polite, transparent, and minimal.

If you remember one rule: don’t create load or harm for the site you read.

  • Check if an official API exists — use it first.
  • Respect robots.txt as a minimum signal (not a legal shield).
  • Rate limit (e.g., 1 request every few seconds) and use caching.
  • Identify yourself with a clear User-Agent if allowed.

Tooling that keeps you out of trouble

The right tooling helps you avoid accidental abuse. You want timeouts, retries with backoff, and clear failure handling so you don’t spam a server.

A practical, safe stack for many tasks is: requests + BeautifulSoup + a simple cache (files or sqlite).

  • Always set timeouts (connect + read).
  • Retry only on safe failures; stop after a small limit.
  • Store the last fetched result and re-use it when possible.
  • Prefer parsing stable page elements, not fragile selectors.

When you should NOT scrape

Sometimes the right answer is: don’t scrape. If the site forbids it, the data is personal/sensitive, or your use-case could harm people, step back and choose a different approach.

If you need data for research or business, ask for permission or use licensed datasets.

  • Avoid scraping accounts, emails, phone numbers, or anything that can identify people.
  • Avoid bypassing paywalls or access controls.
  • Avoid high-frequency scraping of small sites with limited capacity.
  • If you’re unsure, treat it as a “no” and look for an API/dataset.
