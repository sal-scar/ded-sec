SSH from Termux: Remote Access Done Right

Connect to servers securely with SSH keys, sane configs, file transfer, and simple tunnels—without breaking your phone or your server.

Updated: 2026-01-11 • Remote workflow guide

Why you can trust this guide

It follows standard SSH practices used by sysadmins and keeps everything verifiable (keys, configs, and least-privilege access).

1) Install OpenSSH in Termux

Termux can be both an SSH client (connect out) and, if you want, an SSH server (accept connections). Start as a client first.

pkg update -y && pkg upgrade -y
pkg install -y openssh

2) Create a keypair (ed25519)

Use a passphrase. If you lose your phone, the passphrase is your last line of defense.

ssh-keygen -t ed25519 -C "android-termux"
# Copy public key:
cat ~/.ssh/id_ed25519.pub

3) Add your public key to the server

On the server, paste your .pub into ~/.ssh/authorized_keys for the user you will log in as.

# On the server:
mkdir -p ~/.ssh && chmod 700 ~/.ssh
echo 'PASTE_PUBLIC_KEY_HERE' >> ~/.ssh/authorized_keys
chmod 600 ~/.ssh/authorized_keys

4) Make SSH easier with ~/.ssh/config

A config file reduces typing and prevents mistakes (wrong host, wrong user).

nano ~/.ssh/config

Host myserver
  HostName example.com
  User ubuntu
  IdentityFile ~/.ssh/id_ed25519
  IdentitiesOnly yes

5) File transfer: scp vs sftp

Use scp for quick copies, sftp for interactive browsing. Both are encrypted over SSH.

# Copy a file to server:
scp local.txt myserver:/home/ubuntu/

# Interactive:
sftp myserver

6) Simple tunnels (port forwarding)

Tunnels let you access a remote service securely without exposing it to the internet (e.g., a local-only admin panel).

# Forward local port 8080 to remote 127.0.0.1:8000
ssh -L 8080:127.0.0.1:8000 myserver

7) Basic hardening checklist

If you run your own server: disable password login (use keys), keep SSH updated, and limit who can log in.

  • Use keys + passphrase
  • Disable password auth on servers you manage
  • Avoid logging in as root directly
  • Use a firewall / allowlist when possible

SSH keys and basic hardening (do this first)

Password logins are the weakest link. Keys are safer and more convenient once set up correctly.

Hardening checklist:

  • Use key-based auth and disable password auth where possible.
  • Limit who can connect (firewall, allowlist, non-default ports if you must).
  • Keep Termux and OpenSSH updated.
  • Never expose an SSH server to the internet unless you understand the risk.

Useful SSH tricks: config, jump hosts, and port forwarding (safe use)

SSH becomes much easier if you use an SSH config file and name your hosts. It reduces mistakes and makes automation safe.

Port forwarding is powerful—use it for legitimate remote admin and testing, never to bypass access controls.

  • Put common hosts in ~/.ssh/config with Host, HostName, User, IdentityFile.
  • Use ProxyJump (jump host) instead of complicated manual hops.
  • Use local port forwards to access a private web service safely.
  • Close tunnels when you finish; don’t leave them running forever.
