ButSystem.py (Exclusive)

A local-first helper web app you run from your own device — built for private collaboration, awareness, and responsible use.

What it is

ButSystem.py is a self-hosted (you host it) web interface that runs on Termux. It gives you a private dashboard for DMs, group chats, a file vault, profile tools, and optional live location sharing — without relying on a third‑party service by default.

The goal is simple: keep your workflows on your phone, keep your data under your control, and use the tool to organize people and information safely.

Core features

DMs + Groups

Send text, voice, GIFs, and files. Group chats help you coordinate with multiple people in one place.

File Vault

Upload and organize folders/files inside your own ButSystem storage.

Account security

Optional security questions (2FA), trusted-device auto-login, and an admin approval workflow for new accounts.

Live location (opt‑in)

Share your location only when you choose. Stop sharing at any time.

How to run it

ButSystem.py is designed to run on Android through Termux. The exact commands may vary depending on your project folder, but the flow is always: open Termux → go to the script folder → run it → open the local link in your browser.

cd DedSec && python ButSystem.py

If Termux asks for permissions (storage, microphone, etc.), grant only what you actually plan to use.

How to use it

1) Open the link
After you run the script, it prints a local URL. Open it in your phone browser (Chrome / Firefox).
2) Create an account
New accounts can be placed in an approval flow so a trusted admin controls who joins.
3) Use DMs / Groups
Tap a user to open a DM. Create a group to coordinate with multiple people.
4) Use the File Vault
Store private notes, documents, and files inside your own device storage.
5) (Optional) Live location
Use it for safety check-ins and coordination. Always make sure everyone involved consents.

Privacy & safety notes

ButSystem is built for responsible use. Use it only for your own data, your own networks, and people who explicitly agree to participate.

  • Text content can be stored encrypted at rest.
  • Some binary uploads (like files) may be stored unencrypted on-device for speed/compatibility — treat your phone storage as sensitive.
  • If you share a public link (tunnel), you’re exposing the app to the internet. Use strong passwords + 2FA, and share links only with trusted people.

Hypothetical “push‑back” scenarios (legal & ethical)

Our vision is about restoring freedom and protecting people — not harming them. Here are examples of how ButSystem can be used in a defensive, consent‑based way:

Create a private group to collect screenshots, timestamps, and links. Organize evidence and agree on a calm response (reporting, blocking, contacting platform support, or legal advice).

Use a group or DM to share the suspicious URL/message and run safe verification steps from the DedSec tools page (without opening unknown files).

Organize who brings what (food, clothes, supplies), share pickup locations, and keep communication in one place.

Opt-in live location sharing can help trusted friends/family confirm you arrived safely. Stop sharing when you’re done.

If you’re angry, pause. Don’t turn tools into revenge. Focus on safety, evidence, and lawful steps.
