Termux Troubleshooting Manual

Chapter 0: Critical Installation and Repository Issues

Almost all Termux stability issues trace back to an incorrect installation source or an outdated package list. Think of this as trying to build a house on a shaky foundation. Resolve these foundational issues first.

CRITICAL WARNING: The Google Play Store Version is Obsolete!

The version of Termux from the Google Play Store is permanently outdated and unsupported due to changes in Android policy. It cannot update its own packages, leading to constant errors. Using it is like using a web browser from 2010—it's insecure and most things will be broken.

  • Fix: Uninstall any old version completely. Then, install the official, up-to-date version from the F-Droid app store. This also applies to all companion apps like `Termux:API` and `Termux:Styling`. They *must* all be from F-Droid to work together.
  • Problem: Package Update Errors (`404 Not Found`, `repository not signed`, `E: Unable to locate`).

    Cause: This error means Termux can't find the software you're asking for. Think of a repository as a giant software warehouse. This error means your 'map' to the warehouse is wrong (broken link), the warehouse is temporarily closed (mirror issue), or you're using the old Play Store app which has a map to a warehouse that no longer exists.

    Solution:

    1. Step 1: Update & Upgrade (Essential): This command syncs your 'map' with the main server (`pkg update`) and then downloads any new versions of software you already have (`pkg upgrade`).
      pkg update && pkg upgrade -y
    2. Step 2: Change Repository Mirror: A 'mirror' is just an identical copy (a mirror image) of the software warehouse in a different location. If the one you're using is slow or blocked, you can switch to another.
      termux-change-repo

      Select "All mirrors" and choose a different main mirror. Mirrors by Grimler, Alabugin, or A1-Mirrors are good choices. After changing, run `pkg update` again to use the new warehouse.

    3. Step 3: Clear Cache and Rebuild: This is like clearing your web browser's cache. It deletes any old, temporary package files and forces Termux to get a fresh 'map'.
      apt clean && apt update

Chapter 1: Execution, Commands, and File Permissions

When something doesn't run, check the three Ps: Package (Is it installed?), Path (Can Termux find it?), and Permissions (Is it allowed to run?).

  • Error: `bash: git: command not found` (or `wget`, `nano`, `python`, etc.)

    Cause: This is a simple error. You've typed a command, but Termux has no idea what it is. Either the package is not installed, or it's installed in a weird place Termux doesn't check. The `PATH` is Termux's 'address book' of folders where it looks for commands.

    Solution:

    1. Install the package: 99% of the time, this is the fix. The command name is usually the package name.
      pkg install <command-name>
    2. If still not found, check your 'address book' (PATH):
      echo $PATH

      It *must* contain `/data/data/com.termux/files/usr/bin`. If it doesn't, add `export PATH=$PREFIX/bin:$PATH` to your `~/.bashrc` file (your shell's startup instructions).

  • Problem: I don't know the exact name of the package to install.

    Cause: Package names can be non-obvious (e.g., the `convert` command is inside the `imagemagick` package).

    Solution: Search for packages: This lets you search the 'warehouse' for keywords.

    pkg search <query>

    For example, `pkg search "image editor"` will list packages related to image manipulation.

  • Error: `bash: ./myscript.sh: Permission denied` when running a script.

    Cause: In Linux, files have 'permission flags'. A file might be 'readable' or 'writable', but it needs a special 'executable' flag to be run as a program. This is a safety feature. You're trying to 'run' a file that's only marked as 'plain text'.

    Solution: Make the script executable:

    chmod +x ./myscript.sh

    `chmod` means "CHange MODe", and `+x` means "add the eXecutable flag". Also, your script's *very first line* should be a 'shebang' (`#!`) telling the system *what program* to use to run it, like `#!/data/data/com.termux/files/usr/bin/bash`.

  • Error: `Permission denied` when accessing Android storage (SD Card, Downloads, etc.).

    Cause: Termux is an app and needs to ask you for permission to see your files, just like any other app.

    Solution: Setup Storage Access:

    1. Run the setup utility: This will trigger the Android pop-up asking, "Allow Termux to access photos, media, and files?" You *must* tap 'Allow'.
      termux-setup-storage
    2. Crucial Caveat: Do not try to run programs *from* your SD card. Android formats external storage with a `noexec` (no execute) flag. This means it's a 'Read-Only' zone for programs. Always work inside your Termux Home Directory (`~`).

Chapter 2: Networking and DNS Issues

When network requests fail, the problem is rarely Termux itself. It's usually DNS (the internet's 'phonebook'), a blocked server, or your local network.

  • Problem: Termux cannot resolve domain names or download packages (DNS Failure).

    Cause: DNS (Domain Name System) is the internet's phonebook. It turns a friendly name like 'google.com' into a computer-readable IP address like '8.8.8.8'. If your 'phonebook' is broken, you can't find anything. This can be caused by a bad Wi-Fi network, a VPN, or Android's "Private DNS" setting.

    Solution: Diagnose and Fix DNS

    1. Test basic connectivity: `ping 8.8.8.8`. If this works, you *are* connected to the internet.
    2. Test DNS resolution: `ping google.com`. If Step 1 worked but this fails, your 'phonebook' is broken.
    3. Force Google DNS (Most Reliable Fix): This command tells Termux to ignore your device's broken phonebook and *only* use Google's reliable one (`8.8.8.8`).
      rm $PREFIX/etc/resolv.conf && echo "nameserver 8.8.8.8" > $PREFIX/etc/resolv.conf
    4. Check for Interference: Temporarily disable any VPN or Android's "Private DNS" setting and try again.
  • Error: Git clone fails with `ssl certificate problem: unable to get local issuer certificate`.

    Cause: An SSL certificate is a website's 'digital passport' to prove it's secure and legitimate. Your system has a 'passport office' (`ca-certificates`) that holds all the trusted 'stamps'. If your 'passport office' is old, it can't verify new passports.

    Solution: Install and Update CA Certificates: This updates your 'passport office' with all the latest trusted authorities.

    pkg install ca-certificates && update-ca-certificates

Chapter 3: Stability, Background, and API Issues

This chapter handles Android itself interfering with Termux, causing it to crash or processes to be "killed" by the system.

  • Problem: Termux freezing, crashing, or processes being killed unexpectedly (Phantom Processes).

    Cause: Android is *designed* to kill apps running in the background to save battery. It sees Termux as 'just another app' and kills your long-running script, thinking it's a runaway process. This is especially aggressive on Android 10+.

    Solution:

    1. Disable battery optimization: You must go into your *Android Settings* -> Apps -> Termux -> Battery, and set it to "Unrestricted" or "Not optimized." This tells Android, "Termux is special; *never* put this app to sleep".
    2. Acquire a wake lock: This command tells Android "I am actively doing important work, do not kill me!" It keeps the phone's CPU partially awake. Run this *before* your long script.
      termux-wake-lock

      To release it (and save your battery!), press Ctrl+C or run `termux-wake-unlock` in another session.

    3. Use `tmux`: This creates a *virtual* session that stays running *even if you close the Termux app*. You can 'detach' from it, close Termux, and 're-attach' hours later. Essential for servers or long jobs.
      pkg install tmux && tmux new -s persistent-session
  • Error: Termux:API commands not found or not working (e.g., `termux-battery-status`).

    Cause: This is a common point of confusion. You need *two* separate things: the 'keys' (the `termux-api` package *inside* Termux) and the 'car' (the `Termux:API` *app* from F-Droid). One is the tool, the other is the engine that connects to Android. Both must be from F-Droid to have matching signatures.

    Solution:

    1. Install the API package in Termux:
      pkg install termux-api
    2. Install the separate Termux:API app from F-Droid.
    3. Reload Termux settings: This forces Termux to find the new API commands you just enabled.
      termux-reload-settings

Chapter 4: Package-Specific and Advanced Build Errors

Errors when `pip install` (Python) or `npm install` (NodeJS) fail because they need to 'build' software from source code.

  • Error: `pip install` fails with `Error: command 'clang' failed` or dependencies missing.

    Cause: Many Python packages (especially for science or speed) are just 'wrappers' around fast code written in C/C++. When you `pip install` them, pip tries to *build* that C code. It fails because you don't have the 'C factory tools' (a compiler like `clang`, `make`, etc.) installed.

    Solution: Install development tools first:

    pkg install build-essential python-dev

    `build-essential` is a bundle that installs the entire 'C factory' (compiler, etc.). `python-dev` installs the 'Python blueprints' (header files) so the factory knows how to build parts that fit with Python.

  • Error: Pip install fails for packages like `lxml`, `scipy`, or `numpy`.

    Cause: Same as above, but these packages *also* need 'blueprints' for *other* system libraries (e.g., `lxml` needs blueprints for `libxml2`, a 'XML parser' factory).

    Solution: Install prerequisite headers: The `-dev` suffix *always* means 'developer blueprints/headers'. You're installing the blueprints for all the parts your Python package needs to build.

    pkg install libxml2-dev libxslt-dev libblas-dev openblas-dev libjpeg-turbo-dev libpng-dev

    After this, try the `pip install` command again.

  • Error: `Segmentation fault` or `Illegal Instruction` during program execution.

    Cause: This is a fatal crash. It's like trying to run a 64-bit Windows program on a 32-bit computer. Your phone's CPU (its 'brain') is likely 'ARM' (e.g., `aarch64`). You are trying to run a program built for a PC's 'x86' brain. They speak different languages.

    Solution: Use the command `uname -m` to see what 'brain' (architecture) you have. Never copy programs from a PC. Only install software using `pkg install`, which guarantees you get the correct version for your phone's architecture.

Chapter 5: Terminal Display and Configuration

Fixing visual errors, shell startup issues, and keyboard problems.

  • Problem: Changes to `.bashrc` or `.zshrc` are ignored until I restart Termux.

    Cause: Your `.bashrc` file is *only* read when you open a *new* shell session. It's not a 'live' settings file that's constantly monitored.

    Solution: Manually reload the configuration: The `source` command says, "Read and execute the commands in this file *right now* in my *current* session".

    source ~/.bashrc (or source ~/.zshrc)
  • Problem: I can't type special keys (like F-keys, Esc, or Ctrl+C) easily.

    Cause: Your phone keyboard doesn't have a `Ctrl` or `Esc` key. Termux has clever built-in workarounds.

    Solution: Use built-in key combinations and the extra keys view:

    • `CTRL` Key: Press the Volume Down button and then the character key (e.g., `Vol Down` + `C` is Ctrl+C).
    • `ESC` Key: Press the Volume Up button and then the `E` key.
    • Customize Extra Keys: This is the *best* solution. By creating/editing `~/.termux/termux.properties`, you can add a permanent on-screen row of buttons for `CTRL`, `ALT`, `ESC`, `TAB`, and arrow keys. Reload with `termux-reload-settings`.

Chapter 6: Advanced Environment and Execution

Understanding the Termux sandbox, its unique directory structure, and advanced execution utilities.

Termux Key Environment Variables

Termux is a non-root app. All its files live inside its own sandboxed folder, different from a normal PC.

  • `$PREFIX`: The 'Program Files' folder (`/data/data/com.termux/files/usr`). All your installed commands (`bin`), libraries (`lib`), and config files (`etc`) are here.
  • `$HOME`: Your 'Documents' folder (`/data/data/com.termux/files/home`). This is where you live. Your scripts and personal files go here. It's also known by the shortcut `~`.
  • Error: `sudo: command not found` or need for root permissions.

    Cause: Termux is a normal, 'unprivileged' app. It cannot touch or control the main Android system. `sudo` is a command to *become* the 'root' (admin) user, which Termux is not.

    Solution: Use Proot-Distro for a 'fake' root environment: This is a brilliant workaround. It creates a 'virtual' Linux system (like Ubuntu) *inside* Termux. Within this *virtual* system, you *are* the 'root' user. This lets you use tools like `sudo` and `apt` as if you were on a real server, but you are still safely sandboxed from your main Android OS.

    1. Install proot-distro and a Linux distribution:
      pkg install proot-distro && proot-distro install ubuntu
    2. Login to the distro:
      proot-distro login ubuntu
  • Problem: Program starts and immediately fails silently (or `No such file or directory`).

    Cause: The program is failing for a hidden reason. It might be looking for a file that doesn't exist, or trying to do something Android blocks.

    Solution: Trace System Calls with `strace`: `strace` is a powerful debugger. It's like attaching a 'black box' recorder to the program. It prints *every single thing* the program tries to do (every file it tries to open, every system function it calls) right up until the moment it crashes. This log will almost always show you the *exact* "Permission denied" or "No such file" error that caused the problem.

    pkg install strace && strace -f <failing-command>

Chapter 7: Customization and Graphical Environment (X11/VNC)

Installing alternative shells and setting up a full graphical desktop (like Windows or macOS) inside Termux.

  • Problem: I want better features and customization than the default Bash shell.

    Cause: Bash is the 'Toyota Camry' of shells—reliable, basic, and gets the job done. Shells like `zsh` and `fish` are like 'Teslas'—they have smart autocompletion (it suggests commands as you type), syntax highlighting (commands turn different colors), and are much more powerful.

    Solution: Install Zsh/Fish and Switch Shells:

    1. Install the package: `pkg install zsh` or `pkg install fish`
    2. Switch your default shell: This command tells Termux to use your new shell every time it starts.
      chsh -s zsh (or chsh -s fish)

      You must fully close and restart Termux for this change to take effect.

  • Alternative: I want a more stable, persistent graphical desktop (VNC).

    Cause: You want a full desktop, but X11 (another graphical method) can be fragile. VNC (Virtual Network Computing) is a more robust solution. It runs a *complete virtual desktop* in the background. Your VNC Viewer app then connects to it like a 'remote desktop'.

    Solution: Setup a VNC Server:

    1. Install TigerVNC and a desktop: `fluxbox` is a very lightweight and fast desktop manager.
      pkg install tigervnc fluxbox
    2. Start the VNC server:
      vncserver -localhost no

      Set a password on the first run. This starts a server on port 5901. If you disconnect, the desktop *keeps running* in the background, with all your apps open.

    3. Connect with a client: Install a VNC Viewer app on Android (like AVNC from F-Droid), and connect to `127.0.0.1:5901`.

Chapter 8: Data Recovery and Corrupted Installs

The 'last resort' steps for when Termux is completely broken and nothing else works.

  • Problem: Termux is completely broken, or `pkg` and `apt` commands immediately fail (Corrupted Prefix).

    Cause: Critical files in your '$PREFIX' (Program Files) directory are corrupted. This can happen if you force-kill Termux during an upgrade or run a bad script.

    Solution: Backup and Reset (The Nuclear Option):

    1. Step 1: Backup your Home Directory. This saves your personal files. `tar` is like WinZip. This command `c`reates a `z`ipped archive `f`ile named `termux-backup.tar.gz` on your phone's *internal storage* (in the Downloads folder, for example). It backs up everything (`.`) inside your `$HOME` directory.
      tar -czvf /sdcard/termux-backup.tar.gz -C $HOME .
    2. Step 2: Clear App Data. Go to Android Settings -> Apps -> Termux -> Storage and tap "Clear data". This performs a complete factory reset of Termux, deleting *everything*—all packages and all your files.
    3. Step 3: Restore your data. Re-open the (now empty) Termux, run `termux-setup-storage` again, and then extract (`x`) your backup to restore your personal files.
      tar -xzvf /sdcard/termux-backup.tar.gz -C $HOME

Chapter 9: Scripting and Automation

Using Termux's companion apps to automate tasks and integrate with Android.

  • Problem: I want to access my Termux session from my computer.

    Cause: You want to use your big PC keyboard to type commands. By default, Termux doesn't allow remote access.

    Solution: Set up the OpenSSH server: This turns your phone into a mini-server that you can 'log into' from your PC.

    1. Install OpenSSH: `pkg install openssh`
    2. Set a password: Run `passwd`. This is the password you'll use to log in remotely.
    3. Start the SSH server: `sshd`
    4. Find your phone's IP address: `ifconfig` (look for `wlan0`).
    5. Connect from another computer: The default user is `termux` and the port is *always* `8022`.
      ssh termux@<your-phone-ip> -p 8022
  • Problem: I want to run a script when my device boots up.

    Cause: Requires the `Termux:Boot` companion app, whose *only* job is to poke Termux when the phone finishes booting.

    Solution: Use Termux:Boot:

    1. Install the Termux:Boot app from F-Droid.
    2. Create the startup directory: `mkdir -p ~/.termux/boot/`
    3. Create your script: Create any file in this folder (e.g., `~/.termux/boot/start-sshd`) and make it executable (`chmod +x`). Any executable script here will run on boot.
      #!/data/data/com.termux/files/usr/bin/bash termux-wake-lock sshd
  • Problem: I want to run scripts from my Android home screen.

    Cause: This requires the `Termux:Widget` companion app.

    Solution: Use Termux:Widget:

    1. Install the Termux:Widget app from F-Droid.
    2. Create the shortcuts directory: `mkdir -p ~/.shortcuts/`
    3. Place your scripts there: Create a script like `~/.shortcuts/update-all`. For example:
      #!/data/data/com.termux/files/usr/bin/bash pkg update && pkg upgrade -y termux-toast "Updates complete!"
    4. Add the Termux Widget to your home screen. It will show a list of your scripts. Tapping one runs it. `termux-toast` is great here as it pops up a small confirmation message.
  • Problem: How can my scripts interact with Android (e.g., send a notification, read battery)?

    Cause: This requires the `Termux:API` package and app, which are the 'glue' between your scripts and your phone's hardware.

    Solution: Use Termux:API commands:

    1. Ensure both the `termux-api` package and `Termux:API` app are installed from F-Droid.
    2. Example 1: Show a notification from a script:
      termux-notification --title "Backup Complete" --content "Your backup script has finished."
    3. Example 2: Get battery level and show a toast message:
      battery=$(termux-battery-status | grep -oP '(?<="percentage": )[0-9]+') termux-toast "Battery is at $battery%"

Εγχειρίδιο Αντιμετώπισης Προβλημάτων Termux

Κεφάλαιο 0: Κρίσιμα Ζητήματα Εγκατάστασης και Αποθετηρίων

Σχεδόν όλα τα ζητήματα σταθερότητας του Termux οφείλονται σε λανθασμένη πηγή εγκατάστασης ή σε παρωχημένη λίστα πακέτων. Σκεφτείτε το σαν να προσπαθείτε να χτίσετε ένα σπίτι σε σαθρά θεμέλια. Λύστε πρώτα αυτά τα θεμελιώδη ζητήματα.

ΚΡΙΣΙΜΗ ΠΡΟΕΙΔΟΠΟΙΗΣΗ: Η Έκδοση του Google Play Store είναι Παρωχημένη!

Η έκδοση του Termux από το Google Play Store είναι μόνιμα παρωχημένη και μη υποστηριζόμενη λόγω αλλαγών στην πολιτική του Android. Δεν μπορεί να ενημερώσει τα δικά της πακέτα, οδηγώντας σε συνεχή σφάλματα. Το να τη χρησιμοποιείτε είναι σαν να χρησιμοποιείτε έναν web browser του 2010—είναι ανασφαλής και τα περισσότερα πράγματα δεν θα λειτουργούν.

  • Διόρθωση: Απεγκαταστήστε πλήρως οποιαδήποτε παλιά έκδοση. Στη συνέχεια, εγκαταστήστε την επίσημη, ενημερωμένη έκδοση από το app store F-Droid. Αυτό ισχύει και για όλες τις συνοδευτικές εφαρμογές όπως `Termux:API` και `Termux:Styling`. Πρέπει *όλες* να προέρχονται από το F-Droid για να λειτουργούν σωστά μαζί.
  • Πρόβλημα: Σφάλματα Ενημέρωσης Πακέτων (`404 Not Found`, `repository not signed`, `E: Unable to locate`).

    Αιτία: Αυτό το σφάλμα σημαίνει ότι το Termux δεν μπορεί να βρει το λογισμικό που ζητάτε. Σκεφτείτε ένα αποθετήριο (repository) σαν μια γιγάντια αποθήκη λογισμικού. Αυτό το σφάλμα σημαίνει ότι ο «χάρτης» σας προς την αποθήκη είναι λάθος (χαλασμένος σύνδεσμος), η αποθήκη είναι προσωρινά κλειστή (πρόβλημα mirror) ή χρησιμοποιείτε την παλιά εφαρμογή του Play Store που έχει χάρτη για μια αποθήκη που δεν υπάρχει πλέον.

    Λύση:

    1. Βήμα 1: Ενημέρωση & Αναβάθμιση (Απαραίτητο): Αυτή η εντολή συγχρονίζει τον «χάρτη» σας με τον κεντρικό διακομιστή (`pkg update`) και στη συνέχεια κατεβάζει τυχόν νέες εκδόσεις του λογισμικού που ήδη έχετε (`pkg upgrade`).
      pkg update && pkg upgrade -y
    2. Βήμα 2: Αλλαγή Mirror Αποθετηρίου: Ένα «mirror» (καθρέφτης) είναι απλώς ένα πανομοιότυπο αντίγραφο της αποθήκης λογισμικού σε διαφορετική τοποθεσία. Εάν αυτό που χρησιμοποιείτε είναι αργό ή μπλοκαρισμένο, μπορείτε να αλλάξετε σε άλλο.
      termux-change-repo

      Επιλέξτε "All mirrors" και διαλέξτε ένα διαφορετικό κύριο mirror. Τα mirrors από Grimler, Alabugin, ή A1-Mirrors είναι καλές επιλογές. Μετά την αλλαγή, εκτελέστε ξανά `pkg update` για να χρησιμοποιήσετε τη νέα αποθήκη.

    3. Βήμα 3: Εκκαθάριση Προσωρινής Μνήμης και Αναδημιουργία: Αυτό είναι σαν να καθαρίζετε την cache του browser σας. Διαγράφει τυχόν παλιά, προσωρινά αρχεία πακέτων και αναγκάζει το Termux να πάρει έναν φρέσκο «χάρτη».
      apt clean && apt update

Κεφάλαιο 1: Εκτέλεση, Εντολές, και Άδειες Αρχείων

Όταν κάτι δεν εκτελείται, ελέγξτε τα τρία Π: Πακέτο (Είναι εγκατεστημένο;), Προορισμός (PATH - Μπορεί το Termux να το βρει;), και Προνόμια (Permissions - Επιτρέπεται να εκτελεστεί;).

  • Σφάλμα: `bash: git: command not found` (ή `wget`, `nano`, `python`, κ.λπ.)

    Αιτία: Αυτό είναι ένα απλό σφάλμα. Πληκτρολογήσατε μια εντολή, αλλά το Termux δεν έχει ιδέα τι είναι. Είτε το πακέτο δεν είναι εγκατεστημένο, είτε είναι εγκατεστημένο σε ένα περίεργο μέρος που το Termux δεν ελέγχει. Το `PATH` είναι το «βιβλίο διευθύνσεων» του Termux με τους φακέλους όπου ψάχνει για εντολές.

    Λύση:

    1. Εγκαταστήστε το πακέτο: Στο 99% των περιπτώσεων, αυτή είναι η λύση. Το όνομα της εντολής είναι συνήθως το όνομα του πακέτου.
      pkg install <όνομα-εντολής>
    2. Εάν εξακολουθεί να μην βρίσκεται, ελέγξτε το «βιβλίο διευθύνσεών» σας (PATH):
      echo $PATH

      Πρέπει *οπωσδήποτε* να περιέχει `/data/data/com.termux/files/usr/bin`. Εάν όχι, προσθέστε `export PATH=$PREFIX/bin:$PATH` στο αρχείο `~/.bashrc` (οι οδηγίες εκκίνησης του shell σας).

  • Πρόβλημα: Δεν γνωρίζω το ακριβές όνομα του πακέτου προς εγκατάσταση.

    Αιτία: Τα ονόματα πακέτων μπορεί μερικές φορές να μην είναι προφανή (π.χ., η εντολή `convert` βρίσκεται μέσα στο πακέτο `imagemagick`).

    Λύση: Αναζήτηση πακέτων: Αυτό σας επιτρέπει να ψάξετε στην «αποθήκη» για λέξεις-κλειδιά.

    pkg search <ερώτημα>

    Για παράδειγμα, το `pkg search "image editor"` θα εμφανίσει πακέτα που σχετίζονται με την επεξεργασία εικόνας.

  • Σφάλμα: `bash: ./myscript.sh: Permission denied` κατά την εκτέλεση ενός script.

    Αιτία: Στο Linux, τα αρχεία έχουν «σημαίες αδειών». Ένα αρχείο μπορεί να είναι «αναγνώσιμο» ή «εγγράψιμο», αλλά χρειάζεται μια ειδική «εκτελέσιμη» σημαία για να εκτελεστεί ως πρόγραμμα. Είναι ένα μέτρο ασφαλείας. Προσπαθείτε να «εκτελέσετε» ένα αρχείο που είναι απλώς «απλό κείμενο».

    Λύση: Κάντε το script εκτελέσιμο:

    chmod +x ./myscript.sh

    Το `chmod` σημαίνει "CHange MODe" (Άλλαξε Κατάσταση), και το `+x` σημαίνει "πρόσθεσε την eXecutable (Εκτελέσιμη) σημαία". Επίσης, η *πρώτη γραμμή* του script σας πρέπει να είναι ένα «shebang» (`#!`) που λέει στο σύστημα *ποιο πρόγραμμα* να χρησιμοποιήσει για να το εκτελέσει, π.χ. `#!/data/data/com.termux/files/usr/bin/bash`.

  • Σφάλμα: `Permission denied` κατά την πρόσβαση στον χώρο αποθήκευσης του Android (SD Card, Downloads, κ.λπ.).

    Αιτία: Το Termux είναι μια εφαρμογή και πρέπει να σας ζητήσει άδεια για να δει τα αρχεία σας, όπως κάθε άλλη εφαρμογή.

    Λύση: Ρύθμιση Πρόσβασης Αποθήκευσης:

    1. Εκτελέστε το βοηθητικό πρόγραμμα ρύθμισης: Αυτό θα εμφανίσει το pop-up του Android που ρωτά: "Να επιτραπεί στην εφαρμογή Termux η πρόσβαση σε φωτογραφίες, πολυμέσα και αρχεία;" Πρέπει *οπωσδήποτε* να πατήσετε 'Να επιτραπεί'.
      termux-setup-storage
    2. Κρίσιμη Επιφύλαξη: Μην προσπαθείτε να εκτελέσετε προγράμματα *από* την κάρτα SD σας. Το Android μορφοποιεί τον εξωτερικό χώρο αποθήκευσης με μια σημαία `noexec` (όχι εκτέλεση). Αυτό σημαίνει ότι είναι μια ζώνη 'Μόνο για Ανάγνωση' για προγράμματα. Να δουλεύετε πάντα μέσα στον αρχικό κατάλογο του Termux (`~`).

Κεφάλαιο 2: Θέματα Δικτύωσης και DNS

Όταν αποτυγχάνουν οι αιτήσεις δικτύου, το πρόβλημα σπάνια είναι το ίδιο το Termux. Συνήθως είναι το DNS (ο «τηλεφωνικός κατάλογος» του διαδικτύου), ένας μπλοκαρισμένος διακομιστής ή το τοπικό σας δίκτυο.

  • Πρόβλημα: Το Termux δεν μπορεί να επιλύσει ονόματα τομέα ή να κατεβάσει πακέτα (Αποτυχία DNS).

    Αιτία: Το DNS (Domain Name System) είναι ο τηλεφωνικός κατάλογος του διαδικτύου. Μετατρέπει ένα φιλικό όνομα όπως 'google.com' σε μια διεύθυνση IP που καταλαβαίνουν οι υπολογιστές, όπως '8.8.8.8'. Αν ο «τηλεφωνικός κατάλογος» σας είναι χαλασμένος, δεν μπορείτε να βρείτε κανέναν. Αυτό μπορεί να προκληθεί από ένα κακό δίκτυο Wi-Fi, ένα VPN ή τη ρύθμιση "Private DNS" του Android.

    Λύση: Διάγνωση και Διόρθωση DNS

    1. Δοκιμή βασικής συνδεσιμότητας: `ping 8.8.8.8`. Αν αυτό λειτουργεί, *είστε* συνδεδεμένοι στο διαδίκτυο.
    2. Δοκιμή επίλυσης DNS: `ping google.com`. Αν το Βήμα 1 λειτούργησε αλλά αυτό αποτυγχάνει, ο «τηλεφωνικός κατάλογος» σας είναι χαλασμένος.
    3. Επιβολή Google DNS (Πιο Αξιόπιστη Διόρθωση): Αυτή η εντολή λέει στο Termux να αγνοήσει τον χαλασμένο τηλεφωνικό κατάλογο της συσκευής σας και να χρησιμοποιεί *μόνο* τον αξιόπιστο της Google (`8.8.8.8`).
      rm $PREFIX/etc/resolv.conf && echo "nameserver 8.8.8.8" > $PREFIX/etc/resolv.conf
    4. Έλεγχος για Παρεμβολές: Απενεργοποιήστε προσωρινά οποιοδήποτε VPN ή τη ρύθμιση "Private DNS" του Android και δοκιμάστε ξανά.
  • Σφάλμα: Το Git clone αποτυγχάνει με `ssl certificate problem: unable to get local issuer certificate`.

    Αιτία: Ένα πιστοποιητικό SSL είναι το «ψηφιακό διαβατήριο» ενός ιστότοπου για να αποδείξει ότι είναι ασφαλής και νόμιμος. Το σύστημά σας διαθέτει ένα «γραφείο διαβατηρίων» (`ca-certificates`) που κατέχει όλες τις έμπιστες «σφραγίδες». Αν το «γραφείο διαβατηρίων» σας είναι παλιό, δεν μπορεί να επαληθεύσει νέα διαβατήρια.

    Λύση: Εγκατάσταση και Ενημέρωση Πιστοποιητικών CA: Αυτό ενημερώνει το «γραφείο διαβατηρίων» σας με όλες τις τελευταίες έμπιστες αρχές.

    pkg install ca-certificates && update-ca-certificates

Κεφάλαιο 3: Ζητήματα Σταθερότητας, Φόντου και API

Αυτό το κεφάλαιο ασχολείται με το ίδιο το Android που παρεμβαίνει στο Termux, προκαλώντας το να κλείνει ή οι διεργασίες του να "σκοτώνονται" από το σύστημα.

  • Πρόβλημα: Το Termux παγώνει, κολλάει ή οι διεργασίες τερματίζονται απροσδόκητα (Διεργασίες Φάντασμα).

    Αιτία: Το Android είναι *σχεδιασμένο* να σκοτώνει εφαρμογές που τρέχουν στο παρασκήνιο για εξοικονόμηση μπαταρίας. Βλέπει το Termux ως «απλώς άλλη μια εφαρμογή» και σκοτώνει το script σας που τρέχει για ώρα, νομίζοντας ότι είναι μια διεργασία που ξέφυγε. Αυτό είναι ιδιαίτερα επιθετικό σε Android 10+.

    Λύση:

    1. Απενεργοποιήστε τη βελτιστοποίηση μπαταρίας: Πρέπει να πάτε στις *Ρυθμίσεις Android* -> Εφαρμογές -> Termux -> Μπαταρία και να το ορίσετε σε "Χωρίς περιορισμούς" (Unrestricted) ή "Μη βελτιστοποιημένο". Αυτό λέει στο Android, "Το Termux είναι ξεχωριστό, *ποτέ* μην βάλεις αυτήν την εφαρμογή για ύπνο".
    2. Αποκτήστε κλείδωμα αφύπνισης (wake lock): Αυτή η εντολή λέει στο Android "Κάνω ενεργά σημαντική δουλειά, μη με σκοτώσεις!" Κρατάει τη CPU του τηλεφώνου μερικώς ξύπνια. Εκτελέστε το *πριν* από το μεγάλο script σας.
      termux-wake-lock

      Για να το απελευθερώσετε (και να σώσετε την μπαταρία σας!), πατήστε Ctrl+C ή εκτελέστε `termux-wake-unlock` σε άλλη συνεδρία.

    3. Χρησιμοποιήστε `tmux`: Αυτό δημιουργεί μια *εικονική* συνεδρία που παραμένει σε λειτουργία *ακόμα κι αν κλείσετε την εφαρμογή Termux*. Μπορείτε να «αποσυνδεθείτε» (detach) από αυτήν, να κλείσετε το Termux και να «επανασυνδεθείτε» (re-attach) ώρες αργότερα. Απαραίτητο για servers ή εργασίες που διαρκούν πολύ.
      pkg install tmux && tmux new -s persistent-session
  • Σφάλμα: Οι εντολές Termux:API δεν βρέθηκαν ή δεν λειτουργούν (π.χ., `termux-battery-status`).

    Αιτία: Αυτό είναι ένα συνηθισμένο σημείο σύγχυσης. Χρειάζεστε *δύο* ξεχωριστά πράγματα: τα «κλειδιά» (το πακέτο `termux-api` *μέσα* στο Termux) και το «αυτοκίνητο» (την *εφαρμογή* `Termux:API` από το F-Droid). Το ένα είναι το εργαλείο, το άλλο είναι η μηχανή που συνδέεται με το Android. Και τα δύο πρέπει να είναι από το F-Droid για να έχουν ταιριαστές υπογραφές.

    Λύση:

    1. Εγκαταστήστε το πακέτο API στο Termux:
      pkg install termux-api
    2. Εγκαταστήστε την ξεχωριστή εφαρμογή Termux:API από το F-Droid.
    3. Επαναφόρτωση ρυθμίσεων Termux: Αυτό αναγκάζει το Termux να βρει τις νέες εντολές API που μόλις ενεργοποιήσατε.
      termux-reload-settings

Κεφάλαιο 4: Ζητήματα Συγκεκριμένων Πακέτων και Σύνθετα Σφάλματα Δόμησης

Σφάλματα όταν το `pip install` (Python) ή το `npm install` (NodeJS) αποτυγχάνουν επειδή πρέπει να «χτίσουν» λογισμικό από τον πηγαίο κώδικα.

  • Σφάλμα: Το `pip install` αποτυγχάνει με `Error: command 'clang' failed` ή λείπουν εξαρτήσεις.

    Αιτία: Πολλά πακέτα Python (ειδικά για επιστήμη ή ταχύτητα) είναι απλώς «περιτυλίγματα» γύρω από γρήγορο κώδικα γραμμένο σε C/C++. Όταν κάνετε `pip install`, το pip προσπαθεί να *χτίσει* αυτόν τον κώδικα C. Αποτυγχάνει επειδή δεν έχετε εγκατεστημένα τα «εργαλεία του εργοστασίου C» (έναν compiler όπως το `clang`, `make`, κ.λπ.).

    Λύση: Εγκαταστήστε πρώτα τα εργαλεία ανάπτυξης:

    pkg install build-essential python-dev

    Το `build-essential` είναι ένα πακέτο που εγκαθιστά ολόκληρο το «εργοστάσιο C» (compiler, κ.λπ.). Το `python-dev` εγκαθιστά τα «αρχιτεκτονικά σχέδια» (header files) ειδικά για την Python, ώστε το εργοστάσιο να ξέρει πώς να χτίσει εξαρτήματα που ταιριάζουν με την Python.

  • Σφάλμα: Το Pip install αποτυγχάνει για πακέτα όπως `lxml`, `scipy`, ή `numpy`.

    Αιτία: Το ίδιο με παραπάνω, αλλά αυτά τα πακέτα χρειάζονται *επίσης* «αρχιτεκτονικά σχέδια» για *άλλες* βιβλιοθήκες συστήματος (π.χ., το `lxml` χρειάζεται σχέδια για το `libxml2`, ένα «εργοστάσιο ανάλυσης XML»).

    Λύση: Εγκαταστήστε τις προαπαιτούμενες κεφαλίδες (headers): Το επίθημα `-dev` *πάντα* σημαίνει «αρχιτεκτονικά σχέδια/headers προγραμματιστή». Εγκαθιστάτε τα σχέδια για όλα τα μέρη που χρειάζεται το πακέτο Python για να χτιστεί.

    pkg install libxml2-dev libxslt-dev libblas-dev openblas-dev libjpeg-turbo-dev libpng-dev

    Μετά από αυτό, δοκιμάστε ξανά την εντολή `pip install`.

  • Σφάλμα: `Segmentation fault` ή `Illegal Instruction` κατά την εκτέλεση προγράμματος.

    Αιτία: Αυτό είναι ένα μοιραίο σφάλμα. Είναι σαν να προσπαθείτε να εκτελέσετε ένα πρόγραμμα 64-bit των Windows σε έναν υπολογιστή 32-bit. Η CPU του τηλεφώνου σας (ο «εγκέφαλός» του) είναι πιθανότατα «ARM» (π.χ., `aarch64`). Προσπαθείτε να εκτελέσετε ένα πρόγραμμα φτιαγμένο για τον «εγκέφαλο» ενός υπολογιστή («x86»). Μιλούν διαφορετικές γλώσσες.

    Λύση: Χρησιμοποιήστε την εντολή `uname -m` για να δείτε τι «εγκέφαλο» (αρχιτεκτονική) έχετε. Ποτέ μην αντιγράφετε προγράμματα από υπολογιστή. Εγκαθιστάτε λογισμικό μόνο χρησιμοποιώντας το `pkg install`, το οποίο εγγυάται ότι παίρνετε τη σωστή έκδοση για την αρχιτεκτονική του τηλεφώνου σας.

Κεφάλαιο 5: Οθόνη Τερματικού και Διαμόρφωση

Διόρθωση οπτικών σφαλμάτων, ζητημάτων εκκίνησης κελύφους και προβλημάτων πληκτρολογίου.

  • Πρόβλημα: Οι αλλαγές σε `.bashrc` ή `.zshrc` αγνοούνται μέχρι να επανεκκινήσω το Termux.

    Αιτία: Το αρχείο `.bashrc` διαβάζεται *μόνο* όταν ανοίγετε μια *νέα* συνεδρία shell. Δεν είναι ένα «ζωντανό» αρχείο ρυθμίσεων που παρακολουθείται συνεχώς.

    Λύση: Επαναφόρτωση της διαμόρφωσης χειροκίνητα: Η εντολή `source` λέει, "Διάβασε και εκτέλεσε τις εντολές σε αυτό το αρχείο *αμέσως τώρα* στην *τρέχουσα* συνεδρία μου".

    source ~/.bashrc (ή source ~/.zshrc)
  • Πρόβλημα: Δεν μπορώ να πληκτρολογήσω εύκολα ειδικά πλήκτρα (όπως F-keys, Esc, ή Ctrl+C).

    Αιτία: Το πληκτρολόγιο του τηλεφώνου σας δεν διαθέτει πλήκτρο `Ctrl` ή `Esc`. Το Termux έχει έξυπνους, ενσωματωμένους τρόπους αντιμετώπισης.

    Λύση: Χρησιμοποιήστε ενσωματωμένους συνδυασμούς πλήκτρων και την προβολή επιπλέον πλήκτρων:

    • Πλήκτρο `CTRL`: Πατήστε το πλήκτρο Μείωσης Έντασης και, στη συνέχεια, το πλήκτρο χαρακτήρα (π.χ., `Μείωση Έντασης` + `C` είναι Ctrl+C).
    • Πλήκτρο `ESC`: Πατήστε το πλήκτρο Αύξησης Έντασης και, στη συνέχεια, το πλήκτρο `E`.
    • Προσαρμογή Επιπλέον Πλήκτρων: Αυτή είναι η *καλύτερη* λύση. Δημιουργώντας/επεξεργάζοντας το `~/.termux/termux.properties`, μπορείτε να προσθέσετε μια μόνιμη σειρά κουμπιών στην οθόνη για `CTRL`, `ALT`, `ESC`, `TAB` και τα βελάκια. Επαναφορτώστε με `termux-reload-settings`.

Κεφάλαιο 6: Προηγμένο Περιβάλλον και Εκτέλεση

Κατανόηση του sandbox του Termux, της μοναδικής του δομής καταλόγου και των προηγμένων βοηθητικών προγραμμάτων εκτέλεσης.

Βασικές Μεταβλητές Περιβάλλοντος Termux

Το Termux είναι μια εφαρμογή χωρίς root. Όλα τα αρχεία του ζουν μέσα στον δικό τους φάκελο sandbox, διαφορετικά από έναν κανονικό υπολογιστή.

  • `$PREFIX`: Ο φάκελος "Program Files" (`/data/data/com.termux/files/usr`). Όλες οι εγκατεστημένες εντολές σας (`bin`), οι βιβλιοθήκες (`lib`) και τα αρχεία ρυθμίσεων (`etc`) βρίσκονται εδώ.
  • `$HOME`: Ο φάκελος "Τα Έγγραφά μου" (`/data/data/com.termux/files/home`). Εδώ είναι ο προσωπικός σας χώρος. Τα scripts και τα προσωπικά σας αρχεία πηγαίνουν εδώ. Είναι επίσης γνωστός με τη συντόμευση `~`.
  • Σφάλμα: `sudo: command not found` ή ανάγκη για δικαιώματα root.

    Αιτία: Το Termux είναι μια κανονική εφαρμογή χωρίς «προνόμια». Δεν μπορεί να αγγίξει ή να ελέγξει το κύριο σύστημα Android. Η εντολή `sudo` είναι για να γίνεις ο χρήστης 'root' (διαχειριστής), κάτι που το Termux δεν είναι.

    Λύση: Χρησιμοποιήστε Proot-Distro για ένα «ψεύτικο» περιβάλλον root: Αυτή είναι μια εξαιρετική λύση. Δημιουργεί ένα «εικονικό» σύστημα Linux (όπως το Ubuntu) *μέσα* στο Termux. Μέσα σε αυτό το *εικονικό* σύστημα, *είστε* ο χρήστης 'root'. Αυτό σας επιτρέπει να χρησιμοποιείτε εργαλεία όπως το `sudo` και το `apt` σαν να ήσασταν σε έναν πραγματικό διακομιστή, αλλά παραμένετε ασφαλείς στο sandbox σας, μακριά από το κύριο λειτουργικό σύστημα Android.

    1. Εγκαταστήστε proot-distro και μια διανομή Linux:
      pkg install proot-distro && proot-distro install ubuntu
    2. Σύνδεση στη διανομή:
      proot-distro login ubuntu
  • Πρόβλημα: Το πρόγραμμα ξεκινά και αποτυγχάνει αμέσως σιωπηλά (ή `No such file or directory`).

    Αιτία: Το πρόγραμμα αποτυγχάνει για κρυφό λόγο. Μπορεί να ψάχνει για ένα αρχείο που δεν υπάρχει ή να προσπαθεί να κάνει κάτι που μπλοκάρει το Android.

    Λύση: Εντοπισμός Κλήσεων Συστήματος με `strace`: Το `strace` είναι ένας ισχυρός αποσφαλματωτής. Είναι σαν να συνδέετε ένα «μαύρο κουτί» καταγραφής στο πρόγραμμα. Τυπώνει *τα πάντα* που προσπαθεί να κάνει το πρόγραμμα (κάθε αρχείο που προσπαθεί να ανοίξει, κάθε λειτουργία συστήματος που καλεί) μέχρι τη στιγμή που καταρρέει. Αυτή η καταγραφή σχεδόν πάντα θα σας δείξει το *ακριβές* σφάλμα "Permission denied" ή "No such file" που προκάλεσε το πρόβλημα.

    pkg install strace && strace -f <εντολή-που-αποτυγχάνει>

Κεφάλαιο 7: Προσαρμογή και Γραφικό Περιβάλλον (X11/VNC)

Εγκατάσταση εναλλακτικών κελυφών (shells) και ρύθμιση μιας πλήρους γραφικής επιφάνειας εργασίας (όπως Windows ή macOS) μέσα στο Termux.

  • Πρόβλημα: Θέλω καλύτερες λειτουργίες και προσαρμογή από το προεπιλεγμένο κέλυφος Bash.

    Αιτία: Το Bash είναι το «Toyota Corolla» των shells—αξιόπιστο, βασικό και κάνει τη δουλειά του. Κελύφη όπως το `zsh` και το `fish` είναι σαν «Tesla»—έχουν έξυπνη αυτόματη συμπλήρωση (προτείνουν εντολές καθώς πληκτρολογείτε), επισήμανση σύνταξης (οι εντολές αλλάζουν χρώματα) και είναι πολύ πιο ισχυρά.

    Λύση: Εγκαταστήστε Zsh/Fish και Αλλάξτε Κέλυφος:

    1. Εγκαταστήστε το πακέτο: `pkg install zsh` ή `pkg install fish`
    2. Αλλάξτε το προεπιλεγμένο κέλυφος σας: Αυτή η εντολή λέει στο Termux να χρησιμοποιεί το νέο σας κέλυφος κάθε φορά που ξεκινά.
      chsh -s zsh (ή chsh -s fish)

      Πρέπει να κλείσετε εντελώς και να επανεκκινήσετε το Termux για να τεθεί σε ισχύ αυτή η αλλαγή.

  • Εναλλακτική: Θέλω μια πιο σταθερή, επίμονη γραφική επιφάνεια εργασίας (VNC).

    Αιτία: Θέλετε μια πλήρη επιφάνεια εργασίας, αλλά το X11 (μια άλλη γραφική μέθοδος) μπορεί να είναι ευαίσθητο. Το VNC (Virtual Network Computing) είναι μια πιο στιβαρή λύση. Εκτελεί μια *πλήρη εικονική επιφάνεια εργασίας* στο παρασκήνιο. Η εφαρμογή VNC Viewer συνδέεται έπειτα σε αυτήν σαν «απομακρυσμένη επιφάνεια εργασίας».

    Λύση: Ρύθμιση Εξυπηρετητή VNC:

    1. Εγκαταστήστε TigerVNC και μια επιφάνεια εργασίας: Το `fluxbox` είναι ένας πολύ ελαφρύς και γρήγορος διαχειριστής επιφάνειας εργασίας.
      pkg install tigervnc fluxbox
    2. Εκκίνηση του εξυπηρετητή VNC:
      vncserver -localhost no

      Ορίστε έναν κωδικό πρόσβασης στην πρώτη εκτέλεση. Αυτό ξεκινά έναν διακομιστή στη θύρα 5901. Αν αποσυνδεθείτε, η επιφάνεια εργασίας *συνεχίζει να τρέχει* στο παρασκήνιο, με όλες τις εφαρμογές σας ανοιχτές.

    3. Σύνδεση με έναν client: Εγκαταστήστε μια εφαρμογή VNC Viewer στο Android (όπως το AVNC από το F-Droid) και συνδεθείτε στο `127.0.0.1:5901`.

Κεφάλαιο 8: Ανάκτηση Δεδομένων και Κατεστραμμένες Εγκαταστάσεις

Τα «έσχατα» βήματα για όταν το Termux είναι εντελώς χαλασμένο και τίποτα άλλο δεν λειτουργεί.

  • Πρόβλημα: Το Termux είναι εντελώς χαλασμένο ή οι εντολές `pkg` και `apt` αποτυγχάνουν αμέσως (Κατεστραμμένο Prefix).

    Αιτία: Κρίσιμα αρχεία στον κατάλογό σας '$PREFIX' (Program Files) είναι κατεστραμμένα. Αυτό μπορεί να συμβεί αν τερματίσετε βίαια το Termux κατά τη διάρκεια μιας αναβάθμισης ή εκτελέσετε ένα κακό script.

    Λύση: Δημιουργία Αντιγράφων Ασφαλείας και Επαναφορά (Η Πυρηνική Επιλογή):

    1. Βήμα 1: Δημιουργία Αντιγράφου Ασφαλείας του Home Directory σας. Αυτό σώζει τα προσωπικά σας αρχεία. Το `tar` είναι σαν το WinZip. Αυτή η εντολή `c` (creates) δημιουργεί ένα `z` (zipped) αρχείο `f` (file) με όνομα `termux-backup.tar.gz` στον *εσωτερικό χώρο αποθήκευσης* του τηλεφώνου σας (π.χ. στον φάκελο Downloads). Δημιουργεί αντίγραφο ασφαλείας για τα πάντα (`.`) μέσα στον κατάλογό σας `$HOME`.
      tar -czvf /sdcard/termux-backup.tar.gz -C $HOME .
    2. Βήμα 2: Εκκαθάριση Δεδομένων Εφαρμογής. Πηγαίνετε στις Ρυθμίσεις Android -> Εφαρμογές -> Termux -> Αποθήκευση και πατήστε "Εκκαθάριση δεδομένων". Αυτό εκτελεί μια πλήρη εργοστασιακή επαναφορά του Termux, διαγράφοντας *τα πάντα*—όλα τα πακέτα και όλα τα αρχεία σας.
    3. Βήμα 3: Επαναφορά των δεδομένων σας. Ανοίξτε ξανά το (τώρα άδειο) Termux, εκτελέστε ξανά `termux-setup-storage` και, στη συνέχεια, εξαγάγετε (`x`) το αντίγραφο ασφαλείας σας για να επαναφέρετε τα προσωπικά σας αρχεία.
      tar -xzvf /sdcard/termux-backup.tar.gz -C $HOME

Κεφάλαιο 9: Scripting και Αυτοματοποίηση

Αξιοποίηση των συνοδευτικών εφαρμογών του Termux για αυτοματοποίηση εργασιών και ενσωμάτωση με το Android.

  • Πρόβλημα: Θέλω να έχω πρόσβαση στη συνεδρία Termux από τον υπολογιστή μου.

    Αιτία: Θέλετε να χρησιμοποιήσετε το μεγάλο πληκτρολόγιο του υπολογιστή σας για να πληκτρολογείτε εντολές. Από προεπιλογή, το Termux δεν επιτρέπει απομακρυσμένη πρόσβαση.

    Λύση: Ρύθμιση του εξυπηρετητή OpenSSH: Αυτό μετατρέπει το τηλέφωνό σας σε έναν μίνι-διακομιστή στον οποίο μπορείτε να «συνδεθείτε» από τον υπολογιστή σας.

    1. Εγκαταστήστε το OpenSSH: `pkg install openssh`
    2. Ορίστε έναν κωδικό πρόσβασης: Εκτελέστε `passwd`. Αυτός είναι ο κωδικός που θα χρησιμοποιήσετε για να συνδεθείτε απομακρυσμένα.
    3. Εκκινήστε τον εξυπηρετητή SSH: `sshd`
    4. Βρείτε τη διεύθυνση IP του τηλεφώνου σας: `ifconfig` (αναζητήστε το `wlan0`).
    5. Σύνδεση από άλλο υπολογιστή: Ο προεπιλεγμένος χρήστης είναι `termux` και η θύρα είναι *πάντα* `8022`.
      ssh termux@<η-ip-του-τηλεφώνου-σας> -p 8022
  • Πρόβλημα: Θέλω να εκτελέσω ένα script όταν εκκινεί η συσκευή μου.

    Αιτία: Απαιτεί την συνοδευτική εφαρμογή `Termux:Boot`, της οποίας η *μοναδική* δουλειά είναι να «ξυπνάει» το Termux όταν το τηλέφωνο τελειώνει την εκκίνηση.

    Λύση: Χρήση Termux:Boot:

    1. Εγκαταστήστε την εφαρμογή Termux:Boot από το F-Droid.
    2. Δημιουργήστε τον κατάλογο εκκίνησης: `mkdir -p ~/.termux/boot/`
    3. Δημιουργήστε το script σας: Δημιουργήστε οποιοδήποτε αρχείο σε αυτόν τον φάκελο (π.χ., `~/.termux/boot/start-sshd`) και κάντε το εκτελέσιμο (`chmod +x`). Οποιοδήποτε εκτελέσιμο script εδώ θα εκτελεστεί κατά την εκκίνηση.
      #!/data/data/com.termux/files/usr/bin/bash termux-wake-lock sshd
  • Πρόβλημα: Θέλω να εκτελώ scripts από την αρχική οθόνη του Android.

    Αιτία: Αυτό απαιτεί την συνοδευτική εφαρμογή `Termux:Widget`.

    Λύση: Χρήση Termux:Widget:

    1. Εγκαταστήστε την εφαρμογή Termux:Widget από το F-Droid.
    2. Δημιουργήστε τον κατάλογο συντομεύσεων: `mkdir -p ~/.shortcuts/`
    3. Τοποθετήστε τα scripts σας εκεί: Δημιουργήστε ένα script όπως `~/.shortcuts/update-all`. Για παράδειγμα:
      #!/data/data/com.termux/files/usr/bin/bash pkg update && pkg upgrade -y termux-toast "Οι ενημερώσεις ολοκληρώθηκαν!"
    4. Προσθέστε το Termux Widget στην αρχική σας οθόνη. Θα δείξει μια λίστα με τα scripts σας. Πατώντας ένα, εκτελείται. Το `termux-toast` είναι εξαιρετικό εδώ, καθώς εμφανίζει ένα μικρό μήνυμα επιβεβαίωσης.
  • Πρόβλημα: Πώς μπορούν τα scripts μου να αλληλεπιδράσουν με το Android (π.χ., αποστολή ειδοποίησης, ανάγνωση μπαταρίας);

    Αιτία: Αυτό απαιτεί το πακέτο και την εφαρμογή `Termux:API`, τα οποία είναι η «κόλλα» μεταξύ των scripts σας και του υλικού (hardware) του τηλεφώνου σας.

    Λύση: Χρήση εντολών Termux:API:

    1. Βεβαιωθείτε ότι τόσο το πακέτο `termux-api` όσο και η εφαρμογή `Termux:API` είναι εγκατεστημένα από το F-Droid.
    2. Παράδειγμα 1: Εμφάνιση ειδοποίησης από ένα script:
      termux-notification --title "Το αντίγραφο ασφαλείας ολοκληρώθηκε" --content "Το script αντιγράφων ασφαλείας σας ολοκληρώθηκε."
    3. Παράδειγμα 2: Λήψη επιπέδου μπαταρίας και εμφάνιση μηνύματος toast:
      battery=$(termux-battery-status | grep -oP '(?<="percentage": )[0-9]+') termux-toast "Η μπαταρία είναι στο $battery%"
