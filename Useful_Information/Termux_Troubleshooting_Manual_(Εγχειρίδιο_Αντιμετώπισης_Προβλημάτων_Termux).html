Termux Troubleshooting Manual

Chapter 0: Critical Installation and Repository Issues

All Termux stability issues usually trace back to an incorrect installation source or an outdated package list. Resolve these foundational issues first.

CRITICAL WARNING: The Google Play Store Version is Obsolete!

The version of Termux from the Google Play Store is permanently outdated and unsupported due to changes in Android policy. It suffers from incorrect signing keys and cannot update its own packages, leading to constant errors. It is mandatory to use the version from F-Droid or the official GitHub releases.

  • Fix: Uninstall any old version completely, then install from F-Droid. This applies to all companion apps like `Termux:API` and `Termux:Styling` as well.
  • Problem: Package Update Errors (`404 Not Found`, `repository not signed`, `E: Unable to locate`).

    Cause: Broken repository link, an issue with the mirror server, or using the wrong app version.

    Solution:

    1. Step 1: Update & Upgrade (Essential):
      pkg update && pkg upgrade -y
    2. Step 2: Change Repository Mirror:
      termux-change-repo

      Select "All mirrors" and choose a different main mirror. Mirrors hosted by Grimler, Alabugin, or A1-Mirrors are generally reliable. After changing, run `pkg update` again.

    3. Step 3: Clear Cache and Rebuild:
      apt clean && apt update
    4. Step 4: Fix Expired Key Error (If applicable): Run `pkg update && pkg upgrade`. If a GPG key error persists, the repository signing key may need to be manually refreshed. Check the official Termux GitHub issues for the most current command.

Chapter 1: Execution, Commands, and File Permissions

When something doesn't run, check the three Ps: Package, Path, and Permissions.

  • Error: `bash: git: command not found` (or `wget`, `nano`, `python`, etc.)

    Cause: The package for the command is not installed, or its path is not in the system's PATH variable.

    Solution:

    1. Install the package:
      pkg install <command-name>
    2. If still not found, verify the standard executable path is in your PATH:
      echo $PATH

      It should contain `/data/data/com.termux/files/usr/bin`. If not, add `export PATH=$PREFIX/bin:$PATH` to your `~/.bashrc`.

  • Problem: I don't know the exact name of the package to install.

    Cause: Package names can sometimes be non-obvious (e.g., `imagemagick` for the `convert` command).

    Solution: Search for packages:

    pkg search <query>

    For example, `pkg search "image editor"` will list packages related to image manipulation.

  • Error: `bash: ./myscript.sh: Permission denied` when running a script.

    Cause: The script file does not have the "execute" permission bit set.

    Solution: Make the script executable:

    chmod +x ./myscript.sh

    Also, ensure your script starts with a shebang line pointing to the correct interpreter, like `#!/data/data/com.termux/files/usr/bin/bash`.

  • Error: `Permission denied` when accessing Android storage (SD Card, Downloads, etc.).

    Cause: Termux does not have Android's storage permission yet.

    Solution: Setup Storage Access:

    1. Run the setup utility:
      termux-setup-storage
    2. Crucial Caveat: Do not run complex tools or install packages on external storage. Android formats external storage (like SD cards) with a `noexec` flag, which prevents binary files from executing. Always operate within the Termux Home Directory (`~`) for reliability.

Chapter 2: Networking and DNS Issues

When network requests fail, the problem is rarely the Termux application itself, but often DNS resolution, a blocked mirror, or local network configuration.

  • Problem: Termux cannot resolve domain names or download packages (DNS Failure).

    Cause: Your device's DNS is not being picked up correctly, the mirror is blocked, or a VPN/Private DNS is interfering.

    Solution: Diagnose and Fix DNS

    1. Test basic connectivity: `ping 8.8.8.8`. If this fails, your device has no internet connection.
    2. Test DNS resolution: `ping google.com`. If the previous step worked but this fails, it's a DNS issue.
    3. Force Google DNS (Most Reliable Fix):
      rm $PREFIX/etc/resolv.conf && echo "nameserver 8.8.8.8" > $PREFIX/etc/resolv.conf
    4. Check for Interference: Temporarily disable any VPN or Android's "Private DNS" setting and try again.
  • Error: Git clone fails with `ssl certificate problem: unable to get local issuer certificate`.

    Cause: Git needs the local certificate store updated to trust HTTPS connections.

    Solution: Install and Update CA Certificates:

    pkg install ca-certificates && update-ca-certificates
  • Problem: Package download fails with connection reset or timeout (Blocked Repository).

    Cause: Your ISP or firewall may be blocking the current Termux repository mirror.

    Solution: Force HTTP over HTTPS for the repository:

    termux-change-repo

    When prompted, select a mirror and then choose the option to use HTTP instead of HTTPS if available. This is less secure but can bypass some firewalls.

Chapter 3: Stability, Background, and API Issues

Addressing unexpected application closures and issues with interacting with the Android system.

  • Problem: Termux freezing, crashing, or processes being killed unexpectedly (Phantom Processes).

    Cause: Aggressive battery optimization settings on Android, especially Android 10+ (Phantom Processes).

    Solution:

    1. Disable battery optimization for Termux in Android settings. For device-specific instructions, consult the website `dontkillmyapp.com`.
    2. Acquire a wake lock for long-running tasks:
      termux-wake-lock

      Run this before your long command. To release it, press Ctrl+C or run `termux-wake-unlock` in another session.

    3. Use `tmux` or `screen` to manage sessions that survive disconnections:
      pkg install tmux && tmux new -s persistent-session
    4. If totally frozen, force-kill all Termux processes:
      pkill -f "termux"
  • Error: Termux:API commands not found or not working (e.g., `termux-battery-status`).

    Cause: Requires two separate components: the Termux package and the separate Android app, and both must be from the same source (F-Droid).

    Solution:

    1. Install the API package in Termux:
      pkg install termux-api
    2. Install the separate Termux:API app from F-Droid or GitHub.
    3. Reload Termux settings: After installing, run `termux-reload-settings` to ensure the API commands are recognized.

Chapter 4: Package-Specific and Advanced Build Errors

Common issues when compiling tools or installing Python, NodeJS, or Ruby dependencies.

  • Error: `pip install` fails with `Error: command 'clang' failed` or dependencies missing.

    Cause: Python packages need to compile C/C++ dependencies (build-essentials), which are not installed by default.

    Solution: Install development tools first:

    pkg install build-essential python-dev

    `build-essential` is a meta-package that includes `clang`, `make`, etc. `python-dev` provides necessary headers.

  • Error: Pip install fails for packages like `lxml`, `scipy`, or `numpy`.

    Cause: These packages require specific headers and libraries (like `libxml2` or `openblas`) that are not included in the default development tools.

    Solution: Install prerequisite headers:

    pkg install libxml2-dev libxslt-dev libblas-dev openblas-dev libjpeg-turbo-dev libpng-dev

    The `-dev` suffix is crucial as it contains the header files needed for compilation. Then re-run the `pip install` command.

  • Error: `npm install` or `gem install` fails to build native extensions.

    Cause: Similar to Python, NodeJS and Ruby packages often need a C/C++ compiler and `make`.

    Solution: Ensure build tools are present:

    pkg install build-essential

    This single package usually resolves most `npm` and `gem` build failures.

  • Error: `dpkg: error processing package <package> (--install): <path>: No such file or directory`.

    Cause: Corrupted package metadata, usually after a failed upgrade or installation.

    Solution: Force a fix and clean up:

    dpkg --configure -a && pkg install --fix-missing && apt-get install -f
  • Error: `Segmentation fault` or `Illegal Instruction` during program execution.

    Cause: The tool is not compiled for your device's CPU architecture (e.g., using an x86 tool on an ARM device).

    Solution: Use the command `uname -m` to verify your architecture and only install packages compatible with that architecture (aarch64, armv7l, or i686). Never copy binaries from a standard Linux PC to Termux; they must be compiled for the Android Bionic C library.

Chapter 5: Terminal Display and Configuration

Fixing visual errors, shell startup issues, and keyboard problems.

  • Problem: Changes to `.bashrc` or `.zshrc` are ignored until I restart Termux.

    Cause: Shell configuration files must be manually "sourced" to apply changes immediately.

    Solution: Manually reload the configuration:

    source ~/.bashrc (or source ~/.zshrc)
  • Problem: Unreadable characters or missing symbols when editing files (Text Encoding).

    Cause: The terminal environment is not configured for Unicode (UTF-8) correctly.

    Solution: Set the correct environment variable in your shell config (`~/.bashrc`):

    export LANG=en_US.UTF-8

    Then reload the shell with `source ~/.bashrc`.

  • Problem: I can't type special keys (like F-keys, Esc, or Ctrl+C) easily.

    Cause: Termux uses volume keys and swipes for special characters.

    Solution: Use built-in key combinations and the extra keys view:

    1. `CTRL` Key: Press the Volume Down button and then the character key (e.g., `Vol Down` + `C` for Ctrl+C).
    2. `ESC` Key: Press the Volume Up button and then the `E` key.
    3. Customize Extra Keys: Create/edit the file `~/.termux/termux.properties`. Add custom keys like this: `extra-keys = [['ESC','/','-','HOME','UP','END'],['TAB','CTRL','ALT','LEFT','DOWN','RIGHT']]`. Reload with `termux-reload-settings`.

Chapter 6: Advanced Environment and Execution

Understanding the Termux sandbox, its unique directory structure, and advanced execution utilities.

Termux Key Environment Variables

Termux is a non-root user-space environment. All tools are installed in a specific location that differs from a standard Linux system.

  • `$PREFIX`: The Termux installation root (`/data/data/com.termux/files/usr`). All binaries and libraries are found here.
  • `$HOME`: The user's home directory (`/data/data/com.termux/files/home`). All personal files and user-installed tools should live here.
  • Error: `sudo: command not found` or need for root permissions.

    Cause: Termux is not a rooted environment by default and cannot use superuser commands.

    Solution: Use Proot-Distro for a full Linux environment:

    1. Install proot-distro and a Linux distribution:
      pkg install proot-distro && proot-distro install ubuntu
    2. Login to the distro:
      proot-distro login ubuntu
    3. Inside the distro, you can use its native package manager as root:
      apt update && apt install neofetch
  • Problem: Program fails with `CANNOT LINK EXECUTABLE... "..." is 32-bit instead of 64-bit` or `LD_PRELOAD` issues.

    Cause: Mixing packages from different sources or trying to run a standard Linux binary that is not compatible with Android's linker (`/system/bin/linker`).

    Solution: Use `termux-exec`:

    pkg install termux-exec && termux-elf-cleaner /path/to/binary

    `termux-exec` automatically rewrites shebangs and handles linker paths. `termux-elf-cleaner` can fix some ELF header issues on incompatible binaries.

  • Problem: Program starts and immediately fails silently (or `No such file or directory`).

    Cause: The program may be attempting a system call that is blocked, missing a dependency, or looking for a file in the wrong path.

    Solution: Trace System Calls with `strace`:

    pkg install strace && strace -f <failing-command>

    `strace` will print a log of all system interactions, often revealing the exact "Permission denied" or path error that caused the crash.

Chapter 7: Customization and Graphical Environment (X11/VNC)

Installing alternative shells and setting up a graphical desktop for X-Window or VNC applications.

  • Problem: I want better features and customization than the default Bash shell.

    Cause: Bash is standard, but shells like Zsh and Fish offer advanced autocompletion, syntax highlighting, and extensibility.

    Solution: Install Zsh/Fish and Switch Shells:

    1. Install the package: `pkg install zsh` or `pkg install fish`
    2. Switch your default shell:
      chsh -s zsh (or chsh -s fish)

      You must restart Termux for this change to take effect.

  • Problem: I need to run a graphical (X-Window) Linux application.

    Cause: Graphical applications require an X Server running on the Android device to render the output.

    Solution: Setup X11 Environment:

    1. Install X11 repository and packages:
      pkg install x11-repo && pkg install termux-x11-nightly fluxbox
    2. Install the separate Termux:X11 app from F-Droid or GitHub.
    3. Configure and run: Open the Termux:X11 app. In Termux, run:
      export DISPLAY=':0' && termux-x11 & && sleep 3 && fluxbox &

      This starts the X11 server and the Fluxbox window manager. You can now run graphical apps like `xterm`.

  • Alternative: I want a more stable, persistent graphical desktop.

    Cause: X11 can be fragile. VNC (Virtual Network Computing) provides a more robust virtual desktop that persists even if the client app disconnects.

    Solution: Setup a VNC Server:

    1. Install TigerVNC and a window manager:
      pkg install tigervnc fluxbox
    2. Start the VNC server:
      vncserver -localhost no

      Set a password on the first run. This starts a server on port 5901.

    3. Connect with a client: Install a VNC Viewer app on Android (like AVNC from F-Droid), and connect to `127.0.0.1:5901`.

Chapter 8: Data Recovery and Corrupted Installs

Last resort steps for fixing persistent issues or recovering a Termux session.

  • Problem: Termux is completely broken, or `pkg` and `apt` commands immediately fail (Corrupted Prefix).

    Cause: Critical files in the `$PREFIX` directory (`/usr`) are corrupted, often by bad scripts or a force-kill during an upgrade.

    Solution: Backup and Reset:

    1. Step 1: Backup your Home Directory. Run `termux-setup-storage` first. Then create a compressed archive:
      tar -czvf /sdcard/termux-backup.tar.gz -C $HOME .

      This saves everything in your home directory to a single file in your phone's internal storage.

    2. Step 2: Clear App Data. Go to Android Settings -> Apps -> Termux -> Storage and "Clear data". This performs a complete factory reset of Termux.
    3. Step 3: Restore your data. After re-opening Termux, run `termux-setup-storage` again and then extract your backup:
      tar -xzvf /sdcard/termux-backup.tar.gz -C $HOME

Chapter 9: Scripting and Automation

Leveraging Termux plugins to automate tasks and integrate with the Android system.

  • Problem: I want to access my Termux session from my computer.

    Cause: By default, Termux does not run an SSH server for remote access.

    Solution: Set up the OpenSSH server:

    1. Install OpenSSH: `pkg install openssh`
    2. Set a password: Run `passwd` to set a password for your user. This is what you'll use to log in.
    3. Start the SSH server: `sshd`
    4. Find your device's IP address: `ifconfig` (look for the `inet` address under `wlan0`).
    5. Connect from another computer: Use an SSH client to connect. The default user is `termux` and the default port is `8022`.
      ssh termux@<your-phone-ip> -p 8022
  • Problem: I want to run a script when my device boots up.

    Cause: Requires the `Termux:Boot` companion app to hook into Android's boot event.

    Solution: Use Termux:Boot:

    1. Install the Termux:Boot app from F-Droid.
    2. Create the startup directory: `mkdir -p ~/.termux/boot/`
    3. Create your script: Create a new file like `~/.termux/boot/start-sshd`. Make it executable with `chmod +x`.
      #!/data/data/com.termux/files/usr/bin/bash
      termux-wake-lock
      sshd

      Any executable script placed in this directory will run on boot.

  • Problem: I want to run scripts from my Android home screen.

    Cause: This requires the `Termux:Widget` companion app.

    Solution: Use Termux:Widget:

    1. Install the Termux:Widget app from F-Droid.
    2. Create the shortcuts directory: `mkdir -p ~/.shortcuts/`
    3. Place your scripts there: Create a script like `~/.shortcuts/update-all`. The script will be run by the widget. For example:
      #!/data/data/com.termux/files/usr/bin/bash
      pkg update && pkg upgrade -y
      termux-toast "Updates complete!"
    4. Add the Termux Widget to your home screen. It will list all executable scripts from the `.shortcuts` directory.
  • Problem: How can my scripts interact with Android (e.g., send a notification, read battery status)?

    Cause: This requires the `Termux:API` package and app, which expose Android functionality to shell commands.

    Solution: Use Termux:API commands:

    1. Ensure both the `termux-api` package and `Termux:API` app are installed from F-Droid.
    2. Example 1: Show a notification from a script:
      termux-notification --title "Backup Complete" --content "Your backup script has finished."
    3. Example 2: Get battery level and show a toast message:
      battery=$(termux-battery-status | grep -oP '(?<="percentage": )[0-9]+')
      termux-toast "Battery is at $battery%"

Final Word on Termux

Termux is a powerful and low-level environment. When troubleshooting, always proceed step-by-step, starting with the correct installation source (F-Droid) and proper permissions. If an error persists, always check the official Termux Wiki or GitHub Issues for community-provided solutions.

Εγχειρίδιο Αντιμετώπισης Προβλημάτων Termux

Κεφάλαιο 0: Κρίσιμα Ζητήματα Εγκατάστασης και Αποθετηρίων

Όλα τα ζητήματα σταθερότητας του Termux συνήθως οφείλονται σε λανθασμένη πηγή εγκατάστασης ή σε παρωχημένη λίστα πακέτων. Επιλύστε πρώτα αυτά τα βασικά προβλήματα.

ΚΡΙΣΙΜΗ ΠΡΟΕΙΔΟΠΟΙΗΣΗ: Η Έκδοση του Google Play Store Είναι Παρωχημένη!

Η έκδοση του Termux από το Google Play Store είναι μόνιμα παρωχημένη και μη υποστηριζόμενη λόγω αλλαγών στην πολιτική του Android. Υποφέρει από λανθασμένα κλειδιά υπογραφής και δεν μπορεί να ενημερώσει τα ίδια της τα πακέτα, οδηγώντας σε συνεχή σφάλματα. Είναι υποχρεωτικό να χρησιμοποιήσετε την έκδοση από το F-Droid ή τις επίσημες κυκλοφορίες του GitHub.

  • Διόρθωση: Απεγκαταστήστε πλήρως οποιαδήποτε παλιά έκδοση, και μετά εγκαταστήστε από το F-Droid. Αυτό ισχύει και για όλες τις συνοδευτικές εφαρμογές όπως `Termux:API` και `Termux:Styling`.
  • Πρόβλημα: Σφάλματα Ενημέρωσης Πακέτων (`404 Not Found`, `repository not signed`, `E: Unable to locate`).

    Αιτία: Κατεστραμμένος σύνδεσμος αποθετηρίου, πρόβλημα με τον διακομιστή mirror, ή χρήση λάθος έκδοσης εφαρμογής.

    Λύση:

    1. Βήμα 1: Ενημέρωση & Αναβάθμιση (Απαραίτητο):
      pkg update && pkg upgrade -y
    2. Βήμα 2: Αλλαγή Διακομιστή Mirror (Repository Mirror):
      termux-change-repo

      Επιλέξτε "All mirrors" και διαλέξτε έναν διαφορετικό κύριο mirror. Οι mirrors που φιλοξενούνται από τους Grimler, Alabugin, ή A1-Mirrors είναι γενικά αξιόπιστοι. Μετά την αλλαγή, εκτελέστε ξανά `pkg update`.

    3. Βήμα 3: Εκκαθάριση Cache και Αναδόμηση:
      apt clean && apt update
    4. Βήμα 4: Διόρθωση Σφάλματος Ληγμένου Κλειδιού (Εάν ισχύει): Εκτελέστε `pkg update && pkg upgrade`. Αν ένα σφάλμα κλειδιού GPG επιμένει, το κλειδί υπογραφής του αποθετηρίου μπορεί να χρειάζεται μη αυτόματη ανανέωση. Ελέγξτε τα επίσημα GitHub issues του Termux για την πιο πρόσφατη εντολή.

Κεφάλαιο 1: Εκτέλεση, Εντολές και Δικαιώματα Αρχείων

Όταν κάτι δεν εκτελείται, ελέγξτε τα τρία Π: Πακέτο, Διαδρομή (Path), και Πρόσβαση (Permissions).

  • Σφάλμα: `bash: git: command not found` (ή `wget`, `nano`, `python`, κλπ.)

    Αιτία: Το πακέτο για την εντολή δεν είναι εγκατεστημένο, ή η διαδρομή του δεν βρίσκεται στη μεταβλητή PATH του συστήματος.

    Λύση:

    1. Εγκαταστήστε το πακέτο:
      pkg install <command-name>
    2. Αν εξακολουθεί να μην βρίσκεται, επαληθεύστε ότι η τυπική εκτελέσιμη διαδρομή είναι στο PATH σας:
      echo $PATH

      Θα πρέπει να περιέχει `/data/data/com.termux/files/usr/bin`. Αν όχι, προσθέστε `export PATH=$PREFIX/bin:$PATH` στο `~/.bashrc` σας.

  • Πρόβλημα: Δεν ξέρω το ακριβές όνομα του πακέτου για εγκατάσταση.

    Αιτία: Τα ονόματα των πακέτων μπορεί μερικές φορές να μην είναι προφανή (π.χ., `imagemagick` για την εντολή `convert`).

    Λύση: Αναζήτηση για πακέτα:

    pkg search <query>

    Για παράδειγμα, το `pkg search "image editor"` θα εμφανίσει πακέτα που σχετίζονται με την επεξεργασία εικόνων.

  • Σφάλμα: `bash: ./myscript.sh: Permission denied` κατά την εκτέλεση ενός script.

    Αιτία: Το αρχείο script δεν έχει οριστεί το δικαίωμα "εκτέλεσης".

    Λύση: Κάντε το script εκτελέσιμο:

    chmod +x ./myscript.sh

    Επίσης, βεβαιωθείτε ότι το script σας ξεκινά με μια γραμμή shebang που δείχνει στον σωστό διερμηνέα, όπως `#!/data/data/com.termux/files/usr/bin/bash`.

  • Σφάλμα: `Permission denied` κατά την πρόσβαση σε αποθηκευτικό χώρο Android (SD Card, Downloads, κλπ.).

    Αιτία: Το Termux δεν έχει ακόμα την άδεια αποθήκευσης του Android.

    Λύση: Ρύθμιση Πρόσβασης Αποθηκευτικού Χώρου:

    1. Εκτελέστε το βοηθητικό πρόγραμμα ρύθμισης:
      termux-setup-storage
    2. Κρίσιμη Επιφύλαξη: Μην εκτελείτε πολύπλοκα εργαλεία ή μην εγκαθιστάτε πακέτα σε εξωτερικό αποθηκευτικό χώρο. Το Android μορφοποιεί τον εξωτερικό χώρο (όπως κάρτες SD) με τη σημαία `noexec`, η οποία εμποδίζει την εκτέλεση δυαδικών αρχείων. Πάντα να λειτουργείτε εντός του Αρχικού Καταλόγου του Termux (`~`) για αξιοπιστία.

Κεφάλαιο 2: Θέματα Δικτύωσης και DNS

Όταν αποτυγχάνουν οι αιτήσεις δικτύου, το πρόβλημα σπάνια είναι η ίδια η εφαρμογή Termux, αλλά συχνά η ανάλυση ονομάτων τομέα (DNS), ένας μπλοκαρισμένος mirror, ή η τοπική διαμόρφωση δικτύου.

  • Πρόβλημα: Το Termux δεν μπορεί να αναλύσει ονόματα τομέα ή να κατεβάσει πακέτα (Αποτυχία DNS).

    Αιτία: Το DNS της συσκευής σας δεν ανιχνεύεται σωστά, ο mirror είναι μπλοκαρισμένος, ή ένα VPN/Ιδιωτικό DNS παρεμβαίνει.

    Λύση: Διάγνωση και Επιδιόρθωση DNS

    1. Ελέγξτε τη βασική συνδεσιμότητα: `ping 8.8.8.8`. Αν αυτό αποτύχει, η συσκευή σας δεν έχει σύνδεση στο διαδίκτυο.
    2. Ελέγξτε την ανάλυση DNS: `ping google.com`. Αν το προηγούμενο βήμα λειτούργησε αλλά αυτό αποτυγχάνει, είναι πρόβλημα DNS.
    3. Επιβολή Google DNS (Πιο Αξιόπιστη Διόρθωση):
      rm $PREFIX/etc/resolv.conf && echo "nameserver 8.8.8.8" > $PREFIX/etc/resolv.conf
    4. Έλεγχος για Παρεμβολές: Απενεργοποιήστε προσωρινά οποιοδήποτε VPN ή τη ρύθμιση "Ιδιωτικό DNS" του Android και δοκιμάστε ξανά.
  • Σφάλμα: Το Git clone αποτυγχάνει με `ssl certificate problem: unable to get local issuer certificate`.

    Αιτία: Το Git χρειάζεται ενημερωμένο τοπικό κατάστημα πιστοποιητικών για να εμπιστεύεται τις συνδέσεις HTTPS.

    Λύση: Εγκατάσταση και Ενημέρωση Πιστοποιητικών CA:

    pkg install ca-certificates && update-ca-certificates
  • Πρόβλημα: Η λήψη πακέτου αποτυγχάνει με επαναφορά σύνδεσης ή timeout (Μπλοκαρισμένο Αποθετήριο).

    Αιτία: Ο πάροχος υπηρεσιών διαδικτύου (ISP) ή το firewall σας ενδέχεται να μπλοκάρει τον τρέχοντα mirror αποθετηρίου Termux.

    Λύση: Επιβολή HTTP αντί για HTTPS για το αποθετήριο:

    termux-change-repo

    Όταν σας ζητηθεί, επιλέξτε έναν mirror και μετά διαλέξτε την επιλογή να χρησιμοποιήσετε HTTP αντί για HTTPS, αν είναι διαθέσιμη. Αυτό είναι λιγότερο ασφαλές αλλά μπορεί να παρακάμψει ορισμένα firewalls.

Κεφάλαιο 3: Θέματα Σταθερότητας, Φόντου και API

Αντιμετώπιση απροσδόκητων κλεισιμάτων εφαρμογών και ζητημάτων αλληλεπίδρασης με το σύστημα Android.

  • Πρόβλημα: Το Termux παγώνει, κρασάρει, ή οι διεργασίες τερματίζονται απροσδόκητα (Phantom Processes).

    Αιτία: Επιθετικές ρυθμίσεις βελτιστοποίησης μπαταρίας στο Android, ειδικά στο Android 10+ (Phantom Processes).

    Λύση:

    1. Απενεργοποιήστε τη βελτιστοποίηση μπαταρίας για το Termux στις ρυθμίσεις του Android. Για οδηγίες ανάλογα με τη συσκευή, συμβουλευτείτε την ιστοσελίδα `dontkillmyapp.com`.
    2. Αποκτήστε wake lock για εργασίες μεγάλης διάρκειας:
      termux-wake-lock

      Εκτελέστε αυτό πριν από τη μακροσκελή εντολή σας. Για να το απελευθερώσετε, πατήστε Ctrl+C ή εκτελέστε `termux-wake-unlock` σε άλλη περίοδο λειτουργίας.

    3. Χρησιμοποιήστε `tmux` ή `screen` για τη διαχείριση περιόδων σύνδεσης που επιβιώνουν από αποσυνδέσεις:
      pkg install tmux && tmux new -s persistent-session
    4. Αν έχει παγώσει εντελώς, τερματίστε βίαια όλες τις διεργασίες Termux:
      pkill -f "termux"
  • Σφάλμα: Οι εντολές Termux:API δεν βρίσκονται ή δεν λειτουργούν (π.χ., `termux-battery-status`).

    Αιτία: Απαιτούνται δύο ξεχωριστά στοιχεία: το πακέτο Termux και η ξεχωριστή εφαρμογή Android, και τα δύο πρέπει να προέρχονται από την ίδια πηγή (F-Droid).

    Λύση:

    1. Εγκαταστήστε το πακέτο API στο Termux:
      pkg install termux-api
    2. Εγκαταστήστε την ξεχωριστή εφαρμογή Termux:API από το F-Droid ή το GitHub.
    3. Επαναφορτώστε τις ρυθμίσεις του Termux: Μετά την εγκατάσταση, εκτελέστε `termux-reload-settings` για να βεβαιωθείτε ότι οι εντολές API αναγνωρίζονται.

Κεφάλαιο 4: Ειδικά Σφάλματα Πακέτων και Σύνθετα Σφάλματα Δόμησης

Κοινά ζητήματα κατά τη μεταγλώττιση εργαλείων ή την εγκατάσταση εξαρτήσεων Python, NodeJS, ή Ruby.

  • Σφάλμα: Το `pip install` αποτυγχάνει με `Error: command 'clang' failed` ή έλλειψη εξαρτήσεων.

    Αιτία: Τα πακέτα Python πρέπει να μεταγλωττίσουν εξαρτήσεις C/C++ (build-essentials), οι οποίες δεν είναι εγκατεστημένες από προεπιλογή.

    Λύση: Εγκαταστήστε πρώτα τα εργαλεία ανάπτυξης:

    pkg install build-essential python-dev

    Το `build-essential` είναι ένα μετα-πακέτο που περιλαμβάνει `clang`, `make`, κλπ. Το `python-dev` παρέχει απαραίτητα headers.

  • Σφάλμα: Το Pip install αποτυγχάνει για πακέτα όπως `lxml`, `scipy`, ή `numpy`.

    Αιτία: Αυτά τα πακέτα απαιτούν συγκεκριμένα headers και βιβλιοθήκες (όπως `libxml2` ή `openblas`) που δεν περιλαμβάνονται στα προεπιλεγμένα εργαλεία ανάπτυξης.

    Λύση: Εγκαταστήστε τις προαπαιτούμενες κεφαλίδες (headers):

    pkg install libxml2-dev libxslt-dev libblas-dev openblas-dev libjpeg-turbo-dev libpng-dev

    Το επίθημα `-dev` είναι κρίσιμο καθώς περιέχει τα αρχεία header που χρειάζονται για τη μεταγλώττιση. Στη συνέχεια, εκτελέστε ξανά την εντολή `pip install`.

  • Σφάλμα: Το `npm install` ή `gem install` αποτυγχάνει στη δημιουργία native extensions.

    Αιτία: Παρόμοια με την Python, τα πακέτα NodeJS και Ruby συχνά χρειάζονται έναν μεταγλωττιστή C/C++ και `make`.

    Λύση: Βεβαιωθείτε ότι υπάρχουν τα εργαλεία δόμησης:

    pkg install build-essential

    Αυτό το μεμονωμένο πακέτο συνήθως επιλύει τις περισσότερες αποτυχίες δόμησης `npm` και `gem`.

  • Σφάλμα: `dpkg: error processing package <package> (--install): <path>: No such file or directory`.

    Αιτία: Κατεστραμμένα μεταδεδομένα πακέτου, συνήθως μετά από μια αποτυχημένη αναβάθμιση ή εγκατάσταση.

    Λύση: Επιβολή διόρθωσης και εκκαθάρισης:

    dpkg --configure -a && pkg install --fix-missing && apt-get install -f
  • Σφάλμα: `Segmentation fault` ή `Illegal Instruction` κατά την εκτέλεση προγράμματος.

    Αιτία: Το εργαλείο δεν έχει μεταγλωττιστεί για την αρχιτεκτονική CPU της συσκευής σας (π.χ., χρήση ενός εργαλείου x86 σε συσκευή ARM).

    Λύση: Χρησιμοποιήστε την εντολή `uname -m` για να επαληθεύσετε την αρχιτεκτονική σας και εγκαταστήστε μόνο πακέτα συμβατά με αυτήν την αρχιτεκτονική (aarch64, armv7l, ή i686). Ποτέ μην αντιγράφετε binaries από έναν τυπικό υπολογιστή Linux στο Termux- πρέπει να μεταγλωττιστούν για τη βιβλιοθήκη Android Bionic C.

Κεφάλαιο 5: Οθόνη Τερματικού και Διαμόρφωση

Διόρθωση οπτικών σφαλμάτων, ζητημάτων εκκίνησης κελύφους και προβλημάτων πληκτρολογίου.

  • Πρόβλημα: Οι αλλαγές σε `.bashrc` ή `.zshrc` αγνοούνται μέχρι να επανεκκινήσω το Termux.

    Αιτία: Τα αρχεία διαμόρφωσης του κελύφους πρέπει να "πηγάζονται" (sourced) χειροκίνητα για να εφαρμοστούν οι αλλαγές άμεσα.

    Λύση: Χειροκίνητη επαναφόρτωση της διαμόρφωσης:

    source ~/.bashrc (ή source ~/.zshrc)
  • Πρόβλημα: Μη αναγνώσιμοι χαρακτήρες ή ελλείποντα σύμβολα κατά την επεξεργασία αρχείων (Κωδικοποίηση Κειμένου).

    Αιτία: Το περιβάλλον τερματικού δεν έχει ρυθμιστεί σωστά για Unicode (UTF-8).

    Λύση: Ορίστε τη σωστή μεταβλητή περιβάλλοντος στη διαμόρφωση του κελύφους σας (`~/.bashrc`):

    export LANG=en_US.UTF-8

    Στη συνέχεια, επαναφορτώστε το κέλυφος με `source ~/.bashrc`.

  • Πρόβλημα: Δεν μπορώ να πληκτρολογήσω εύκολα ειδικά πλήκτρα (όπως F-keys, Esc, ή Ctrl+C).

    Αιτία: Το Termux χρησιμοποιεί τα πλήκτρα έντασης και swipes για ειδικούς χαρακτήρες.

    Λύση: Χρησιμοποιήστε τους ενσωματωμένους συνδυασμούς πλήκτρων και την προβολή επιπλέον πλήκτρων:

    1. `CTRL` Key: Πατήστε το πλήκτρο Μείωσης Έντασης (Volume Down) και στη συνέχεια το πλήκτρο χαρακτήρα (π.χ., `Vol Down` + `C` για Ctrl+C).
    2. `ESC` Key: Πατήστε το πλήκτρο Αύξησης Έντασης (Volume Up) και στη συνέχεια το πλήκτρο `E`.
    3. Προσαρμογή Επιπλέον Πλήκτρων: Δημιουργήστε/επεξεργαστείτε το αρχείο `~/.termux/termux.properties`. Προσθέστε προσαρμοσμένα πλήκτρα όπως: `extra-keys = [['ESC','/','-','HOME','UP','END'],['TAB','CTRL','ALT','LEFT','DOWN','RIGHT']]`. Επαναφορτώστε με `termux-reload-settings`.

Κεφάλαιο 6: Σύνθετο Περιβάλλον και Εκτέλεση

Κατανόηση του Termux sandbox, της μοναδικής δομής καταλόγων του και των προηγμένων βοηθητικών προγραμμάτων εκτέλεσης.

Βασικές Μεταβλητές Περιβάλλοντος Termux

Το Termux είναι ένα ισχυρό περιβάλλον user-space χωρίς root. Όλα τα εργαλεία είναι εγκατεστημένα σε μια συγκεκριμένη θέση που διαφέρει από ένα τυπικό σύστημα Linux.

  • `$PREFIX`: Η ρίζα εγκατάστασης του Termux (`/data/data/com.termux/files/usr`). Όλα τα binaries και οι βιβλιοθήκες βρίσκονται εδώ.
  • `$HOME`: Ο αρχικός κατάλογος του χρήστη (`/data/data/com.termux/files/home`). Όλα τα προσωπικά αρχεία και τα εργαλεία που εγκαθιστά ο χρήστης πρέπει να βρίσκονται εδώ.
  • Σφάλμα: `sudo: command not found` ή ανάγκη για δικαιώματα root.

    Αιτία: Το Termux δεν είναι περιβάλλον rooted από προεπιλογή και δεν μπορεί να χρησιμοποιήσει εντολές superuser.

    Λύση: Χρήση Proot-Distro για ένα πλήρες περιβάλλον Linux:

    1. Εγκατάσταση του proot-distro και μιας διανομής Linux:
      pkg install proot-distro && proot-distro install ubuntu
    2. Σύνδεση στη διανομή:
      proot-distro login ubuntu
    3. Μέσα στη διανομή, μπορείτε να χρησιμοποιήσετε τον εγγενή διαχειριστή πακέτων της ως root:
      apt update && apt install neofetch
  • Πρόβλημα: Το πρόγραμμα αποτυγχάνει με `CANNOT LINK EXECUTABLE... "..." is 32-bit instead of 64-bit` ή θέματα `LD_PRELOAD`.

    Αιτία: Ανάμειξη πακέτων από διαφορετικές πηγές ή προσπάθεια εκτέλεσης ενός τυπικού δυαδικού αρχείου Linux που δεν είναι συμβατό με τον linker του Android (`/system/bin/linker`).

    Λύση: Χρήση `termux-exec`:

    pkg install termux-exec && termux-elf-cleaner /path/to/binary

    Το `termux-exec` ξαναγράφει αυτόματα τα shebangs και διαχειρίζεται τις διαδρομές του linker. Το `termux-elf-cleaner` μπορεί να διορθώσει ορισμένα προβλήματα επικεφαλίδας ELF σε μη συμβατά binaries.

  • Πρόβλημα: Το πρόγραμμα ξεκινά και αποτυγχάνει αμέσως σιωπηλά (ή `No such file or directory`).

    Αιτία: Το πρόγραμμα μπορεί να επιχειρεί μια κλήση συστήματος που είναι μπλοκαρισμένη, λείπει μια εξάρτηση, ή ψάχνει για ένα αρχείο σε λάθος διαδρομή.

    Λύση: Παρακολούθηση Κλήσεων Συστήματος με `strace`:

    pkg install strace && strace -f <failing-command>

    Το `strace` θα εμφανίσει ένα αρχείο καταγραφής όλων των αλληλεπιδράσεων του συστήματος, αποκαλύπτοντας συχνά το ακριβές "Permission denied" ή σφάλμα διαδρομής που προκάλεσε το crash.

Κεφάλαιο 7: Προσαρμογή και Γραφικό Περιβάλλον (X11/VNC)

Εγκατάσταση εναλλακτικών κελυφών και ρύθμιση ενός γραφικού desktop για εφαρμογές X-Window ή VNC.

  • Πρόβλημα: Θέλω καλύτερες λειτουργίες και προσαρμογή από το προεπιλεγμένο κέλυφος Bash.

    Αιτία: Το Bash είναι το πρότυπο, αλλά κελύφη όπως το Zsh και το Fish προσφέρουν προηγμένη αυτόματη συμπλήρωση, επισήμανση σύνταξης και επεκτασιμότητα.

    Λύση: Εγκαταστήστε το Zsh/Fish και Αλλάξτε Κέλυφος:

    1. Εγκαταστήστε το πακέτο: `pkg install zsh` ή `pkg install fish`
    2. Αλλάξτε το προεπιλεγμένο κέλυφος:
      chsh -s zsh (ή chsh -s fish)

      Πρέπει να κάνετε επανεκκίνηση του Termux για να τεθεί σε ισχύ αυτή η αλλαγή.

  • Πρόβλημα: Χρειάζεται να εκτελέσω μια γραφική εφαρμογή Linux (X-Window).

    Αιτία: Οι γραφικές εφαρμογές απαιτούν έναν X Server να εκτελείται στη συσκευή Android για να αποδώσει την έξοδο.

    Λύση: Ρύθμιση Περιβάλλοντος X11:

    1. Εγκαταστήστε το αποθετήριο X11 και πακέτα:
      pkg install x11-repo && pkg install termux-x11-nightly fluxbox
    2. Εγκαταστήστε την ξεχωριστή εφαρμογή Termux:X11 από το F-Droid ή το GitHub.
    3. Διαμόρφωση και εκτέλεση: Ανοίξτε την εφαρμογή Termux:X11. Στο Termux, εκτελέστε:
      export DISPLAY=':0' && termux-x11 & && sleep 3 && fluxbox &

      Αυτό ξεκινά τον διακομιστή X11 και τον διαχειριστή παραθύρων Fluxbox. Τώρα μπορείτε να εκτελέσετε γραφικές εφαρμογές όπως το `xterm`.

  • Εναλλακτική: Θέλω ένα πιο σταθερό, μόνιμο γραφικό περιβάλλον εργασίας.

    Αιτία: Το X11 μπορεί να είναι εύθραυστο. Το VNC (Virtual Network Computing) παρέχει ένα πιο ανθεκτικό εικονικό περιβάλλον εργασίας που παραμένει ενεργό ακόμα και αν η εφαρμογή-πελάτης αποσυνδεθεί.

    Λύση: Ρύθμιση ενός VNC Server:

    1. Εγκατάσταση του TigerVNC και ενός διαχειριστή παραθύρων:
      pkg install tigervnc fluxbox
    2. Εκκίνηση του VNC server:
      vncserver -localhost no

      Ορίστε έναν κωδικό πρόσβασης την πρώτη φορά. Αυτό ξεκινά έναν διακομιστή στη θύρα 5901.

    3. Σύνδεση με έναν client: Εγκαταστήστε μια εφαρμογή VNC Viewer στο Android (όπως το AVNC από το F-Droid), και συνδεθείτε στο `127.0.0.1:5901`.

Κεφάλαιο 8: Ανάκτηση Δεδομένων και Κατεστραμμένες Εγκαταστάσεις

Βήματα έσχατης ανάγκης για τη διόρθωση επίμονων ζητημάτων ή την ανάκτηση μιας συνεδρίας Termux.

  • Πρόβλημα: Το Termux είναι εντελώς χαλασμένο, ή οι εντολές `pkg` και `apt` αποτυγχάνουν αμέσως (Κατεστραμμένο Prefix).

    Αιτία: Κρίσιμα αρχεία στον κατάλογο `$PREFIX` (`/usr`) είναι κατεστραμμένα, συχνά από κακά scripts ή βίαιο τερματισμό κατά τη διάρκεια μιας αναβάθμισης.

    Λύση: Δημιουργία αντιγράφων ασφαλείας και Επαναφορά:

    1. Βήμα 1: Δημιουργία αντιγράφου ασφαλείας του αρχικού σας καταλόγου. Εκτελέστε πρώτα `termux-setup-storage`. Στη συνέχεια, δημιουργήστε ένα συμπιεσμένο αρχείο:
      tar -czvf /sdcard/termux-backup.tar.gz -C $HOME .

      Αυτό αποθηκεύει τα πάντα στον αρχικό σας κατάλογο σε ένα ενιαίο αρχείο στον εσωτερικό χώρο αποθήκευσης του τηλεφώνου σας.

    2. Βήμα 2: Εκκαθάριση Δεδομένων Εφαρμογής. Πηγαίνετε στις Ρυθμίσεις Android -> Εφαρμογές -> Termux -> Αποθηκευτικός χώρος και "Εκκαθάριση δεδομένων". Αυτό εκτελεί μια πλήρη εργοστασιακή επαναφορά του Termux.
    3. Βήμα 3: Επαναφορά των δεδομένων σας. Αφού ανοίξετε ξανά το Termux, εκτελέστε ξανά το `termux-setup-storage` και στη συνέχεια αποσυμπιέστε το αντίγραφο ασφαλείας σας:
      tar -xzvf /sdcard/termux-backup.tar.gz -C $HOME

Κεφάλαιο 9: Scripting και Αυτοματισμός

Αξιοποίηση των plugins του Termux για την αυτοματοποίηση εργασιών και την ενσωμάτωση με το σύστημα Android.

  • Πρόβλημα: Θέλω να έχω πρόσβαση στη συνεδρία μου στο Termux από τον υπολογιστή μου.

    Αιτία: Από προεπιλογή, το Termux δεν εκτελεί διακομιστή SSH για απομακρυσμένη πρόσβαση.

    Λύση: Ρύθμιση του διακομιστή OpenSSH:

    1. Εγκατάσταση του OpenSSH: `pkg install openssh`
    2. Ορισμός κωδικού πρόσβασης: Εκτελέστε `passwd` για να ορίσετε έναν κωδικό πρόσβασης για τον χρήστη σας. Αυτό θα χρησιμοποιήσετε για να συνδεθείτε.
    3. Εκκίνηση του διακομιστή SSH: `sshd`
    4. Εύρεση της διεύθυνσης IP της συσκευής σας: `ifconfig` (αναζητήστε τη διεύθυνση `inet` κάτω από το `wlan0`).
    5. Σύνδεση από άλλον υπολογιστή: Χρησιμοποιήστε έναν πελάτη SSH για να συνδεθείτε. Ο προεπιλεγμένος χρήστης είναι `termux` και η προεπιλεγμένη θύρα είναι `8022`.
      ssh termux@<your-phone-ip> -p 8022
  • Πρόβλημα: Θέλω να εκτελέσω ένα script κατά την εκκίνηση της συσκευής μου.

    Αιτία: Απαιτεί τη συνοδευτική εφαρμογή `Termux:Boot` για να συνδεθεί με το συμβάν εκκίνησης του Android.

    Λύση: Χρήση του Termux:Boot:

    1. Εγκαταστήστε την εφαρμογή Termux:Boot από το F-Droid.
    2. Δημιουργήστε τον κατάλογο εκκίνησης: `mkdir -p ~/.termux/boot/`
    3. Δημιουργήστε το script σας: Δημιουργήστε ένα νέο αρχείο όπως `~/.termux/boot/start-sshd`. Κάντε το εκτελέσιμο με `chmod +x`.
      #!/data/data/com.termux/files/usr/bin/bash
      termux-wake-lock
      sshd

      Οποιοδήποτε εκτελέσιμο script τοποθετηθεί σε αυτόν τον κατάλογο θα εκτελείται κατά την εκκίνηση.

  • Πρόβλημα: Θέλω να εκτελώ scripts από την αρχική οθόνη του Android.

    Αιτία: Αυτό απαιτεί τη συνοδευτική εφαρμογή `Termux:Widget`.

    Λύση: Χρήση του Termux:Widget:

    1. Εγκαταστήστε την εφαρμογή Termux:Widget από το F-Droid.
    2. Δημιουργήστε τον κατάλογο συντομεύσεων: `mkdir -p ~/.shortcuts/`
    3. Τοποθετήστε τα scripts σας εκεί: Δημιουργήστε ένα script όπως `~/.shortcuts/update-all`. Το script θα εκτελεστεί από το widget. Για παράδειγμα:
      #!/data/data/com.termux/files/usr/bin/bash
      pkg update && pkg upgrade -y
      termux-toast "Οι ενημερώσεις ολοκληρώθηκαν!"
    4. Προσθέστε το Termux Widget στην αρχική σας οθόνη. Θα εμφανίσει όλα τα εκτελέσιμα scripts από τον κατάλογο `.shortcuts`.
  • Πρόβλημα: Πώς μπορούν τα scripts μου να αλληλεπιδράσουν με το Android (π.χ., αποστολή ειδοποίησης, ανάγνωση κατάστασης μπαταρίας);

    Αιτία: Αυτό απαιτεί το πακέτο και την εφαρμογή `Termux:API`, τα οποία εκθέτουν τη λειτουργικότητα του Android σε εντολές κελύφους.

    Λύση: Χρήση εντολών Termux:API:

    1. Βεβαιωθείτε ότι τόσο το πακέτο `termux-api` όσο και η εφαρμογή `Termux:API` είναι εγκατεστημένα από το F-Droid.
    2. Παράδειγμα 1: Εμφάνιση ειδοποίησης από ένα script:
      termux-notification --title "Το αντίγραφο ασφαλείας ολοκληρώθηκε" --content "Το script αντιγράφων ασφαλείας σας ολοκληρώθηκε."
    3. Παράδειγμα 2: Λήψη επιπέδου μπαταρίας και εμφάνιση μηνύματος toast:
      battery=$(termux-battery-status | grep -oP '(?<="percentage": )[0-9]+')
      termux-toast "Η μπαταρία είναι στο $battery%"
