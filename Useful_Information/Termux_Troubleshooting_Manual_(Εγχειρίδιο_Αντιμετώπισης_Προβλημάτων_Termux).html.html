Termux Troubleshooting Manual

Chapter 0: Critical Installation and Repository Issues

All Termux stability issues usually trace back to an incorrect installation source or an outdated package list. Resolve these foundational issues first.

CRITICAL WARNING: The Google Play Store Version is Obsolete!

The version of **Termux** from the Google Play Store is permanently outdated and unsupported due to changes in Android policy. It is **mandatory** to use the version from **F-Droid** or the official **GitHub** releases.

  • **Fix:** Uninstall old version, install from **F-Droid**.
  • Problem: Package Update Errors (`404 Not Found`, `repository not signed`, `E: Unable to locate`).

    **Cause:** Broken repository link, an issue with the mirror server, or using the wrong app version.

    **Solution:**

    1. **Step 1: Update & Upgrade (Essential):**
      pkg update && pkg upgrade -y
    2. **Step 2: Change Repository Mirror:**
      termux-change-repo

      Choose a different geographical mirror (e.g., Grimler or Alabugin) and run `pkg update` again.

    3. **Step 3: Clear Cache and Rebuild:**
      apt clean && apt update
    4. **Step 4: Fix Expired Key Error (If applicable):** Run `pkg update && pkg upgrade`. If the error persists, check GitHub for the latest manual key refresh command.

Chapter 1: Execution, Commands, and File Permissions

When something doesn't run, check the three Ps: **Package**, **Path**, and **Permissions**.

  • Error: `bash: git: command not found` (or `wget`, `nano`, `python`, etc.)

    **Cause:** The package for the command is not installed, or its path is not in the system's PATH variable.

    **Solution:**

    1. **Install the package:**
      pkg install
    2. **If still not found**, verify the command is actually in the standard executable paths:
      echo $PATH
  • Error: `Permission denied` when accessing Android storage (SD Card, Downloads, etc.).

    **Cause:** Termux does not have Android's storage permission yet.

    **Solution: Setup Storage Access:**

    1. **Run the setup utility:**
      termux-setup-storage
    2. **Crucial Caveat:** Only run large tools or install complex packages within the **Termux Home Directory** (`~`). External storage paths often cause execution failures (e.g., Git failures).

Chapter 2: Networking and DNS Issues

When network requests fail, the problem is rarely the Termux application itself, but often DNS resolution.

  • Problem: Termux cannot resolve domain names or download packages (DNS Failure).

    **Cause:** Your device's DNS is not being picked up correctly, or the mirror is blocked.

    **Solution: Force Google DNS** (Most Reliable Fix):

    rm $PREFIX/etc/resolv.conf && echo "nameserver 8.8.8.8" > $PREFIX/etc/resolv.conf
  • Error: Git clone fails with `ssl certificate problem: unable to get local issuer certificate`.

    **Cause:** Git needs the local certificate store updated to trust HTTPS connections.

    **Solution: Install and Update CA Certificates:**

    pkg install ca-certificates

Chapter 3: Stability, Background, and API Issues

Addressing unexpected application closures and issues with interacting with the Android system.

  • Problem: Termux freezing, crashing, or processes being killed unexpectedly (Phantom Processes).

    **Cause:** Aggressive battery optimization settings on Android, especially Android 10+ (Phantom Processes).

    **Solution:**

    1. **Disable battery optimization** for Termux in Android settings.
    2. **Use `tmux` or `screen`** to manage sessions that survive disconnections:
      pkg install tmux && tmux new -s persistent-session
    3. **If totally frozen**, force-kill all Termux processes:
      pkill -f "termux"
  • Error: Termux:API commands not found or not working (e.g., `termux-battery-status`).

    **Cause:** Requires two separate components: the Termux package and the separate Android app.

    **Solution:**

    1. **Install the API package in Termux:**
      pkg install termux-api
    2. **Install the separate Termux:API app** from F-Droid or GitHub.

Chapter 4: Package-Specific and Advanced Build Errors

Common issues when compiling tools or installing Python dependencies.

  • Error: `pip install` fails with `Error: command 'gcc' failed` or dependencies missing.

    **Cause:** Python packages need to compile C/C++ dependencies (`build-essentials`).

    **Solution: Install development tools first:**

    pkg install python clang make libcrypt-dev

    These packages provide the compiler and necessary headers for many libraries to build correctly.

  • Error: `dpkg: error processing package (--install): : No such file or directory`.

    **Cause:** Corrupted package metadata, usually after a failed upgrade or installation.

    **Solution: Force a fix and clean up:**

    dpkg --configure -a && pkg install --fix-missing
  • Error: `Segmentation fault` or `Illegal Instruction` during program execution.

    **Cause:** The tool is not compiled for your device's CPU architecture (e.g., using an x86 tool on an ARM device).

    **Solution:** Use the command `uname -m` to verify your architecture and only install packages compatible with that architecture (aarch64, armv7l, or i686).

Chapter 5: Terminal Display and Configuration

Fixing visual errors, shell startup issues, and keyboard problems.

  • Problem: Changes to `.bashrc` or `.zshrc` are ignored until I restart Termux.

    **Cause:** Shell configuration files must be manually "sourced" to apply changes immediately.

    **Solution: Manually reload the configuration:**

    source ~/.bashrc (or source ~/.zshrc)
  • Problem: Unreadable characters or missing symbols when editing files (Text Encoding).

    **Cause:** The terminal environment is not configured for Unicode (UTF-8) correctly.

    **Solution: Set the correct environment variable in your shell config (`~/.bashrc`):**

    export LANG=en_US.UTF-8
  • Problem: I can't type special keys (like F-keys, Esc, or Ctrl+C) easily.

    **Cause:** Termux uses volume keys and swipes for special characters.

    **Solution: Use built-in key combinations and the extra keys view:**

    1. **`CTRL` Key:** Press the **Volume Down** button and then the character key (e.g., `Vol Down` + `C` for Ctrl+C).
    2. **`ESC` Key:** Press the **Volume Up** button and then the `E` key.

Chapter 6: Advanced Environment and Execution

Understanding the Termux sandbox, its unique directory structure, and advanced execution utilities.

Termux Key Environment Variables

Termux is a non-root user-space environment. All tools are installed in a specific location that differs from a standard Linux system.

  • **`$PREFIX`**: The Termux installation root (`/data/data/com.termux/files/usr`). All binaries and libraries are found here.
  • **`$HOME`**: The user's home directory (`/data/data/com.termux/files/home`). All personal files and user-installed tools should live here.
  • Error: `sudo: command not found` or need for root permissions.

    **Cause:** Termux is not a rooted environment by default and cannot use superuser commands.

    **Solution: Use Proot or Fakeroot:**

    1. **To emulate a full Linux root environment (chroot):**
      pkg install proot

      Run a rootless Linux distribution "chroot" using `proot`.

    2. **For building packages requiring root ownership:**
      pkg install fakeroot

      Use `fakeroot` to create archives with specific permissions.

  • Problem: Program starts and immediately fails silently (or `No such file or directory`).

    **Cause:** The program may be attempting a system call (like file access or network setup) that is blocked or missing a dependency, but it's not printing a useful error.

    **Solution: Trace System Calls with `strace`:**

    pkg install strace && strace -f

    `strace` will print a line-by-line log of all system interactions, often revealing the exact "Permission denied" or path error that caused the crash.

Chapter 7: Customization and Graphical Environment (X11)

Installing alternative shells and setting up a basic graphical desktop for X-Window applications.

  • Problem: I want better features and customization than the default Bash shell.

    **Cause:** Bash is the standard, but shells like Zsh offer advanced autocompletion and extensibility via plugins.

    **Solution: Install Zsh and Switch Shells:**

    1. **Install the package:**
      pkg install zsh
    2. **Switch your default shell:**
      chsh -s zsh

      You must reload Termux for this change to take effect.

    3. **Customize the prompt (e.g., in `~/.zshrc`):**
      export PROMPT='%n@%m:%~%# '
  • Problem: I need to run a graphical (X-Window) Linux application.

    **Cause:** Graphical applications require an X Server running on the Android device to render the output.

    **Solution: Setup X11 Environment:**

    1. **Install the X11 repository and client package:**
      pkg install x11-repo && pkg install termux-x11-nightly
    2. **Install the separate Termux:X11 Android app** from F-Droid or GitHub.
    3. **Set the display variable and run an app (e.g., `xeyes`):**
      export DISPLAY=':0' && pkg install xeyes && xeyes

      The Termux:X11 app must be running in the background for this to work.

Final Word on Termux

Termux is a powerful and low-level environment. When troubleshooting, always proceed step-by-step, starting with the correct installation source (F-Droid) and proper permissions. If an error persists, always check the **Termux Wiki** or **GitHub Issues**.
