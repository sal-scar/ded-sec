Termux Troubleshooting Manual

Chapter 0: Critical Installation and Repository Issues

All Termux stability issues usually trace back to an incorrect installation source or an outdated package list. Resolve these foundational issues first.

CRITICAL WARNING: The Google Play Store Version is Obsolete!

The version of **Termux** from the Google Play Store is permanently outdated and unsupported due to changes in Android policy. It is **mandatory** to use the version from **F-Droid** or the official **GitHub** releases.

  • **Fix:** Uninstall old version, install from **F-Droid**.
  • Problem: Package Update Errors (`404 Not Found`, `repository not signed`, `E: Unable to locate`).

    **Cause:** Broken repository link, an issue with the mirror server, or using the wrong app version.

    **Solution:**

    1. **Step 1: Update & Upgrade (Essential):**
      pkg update && pkg upgrade -y
    2. **Step 2: Change Repository Mirror:**
      termux-change-repo

      Choose a different geographical mirror (e.g., Grimler or Alabugin) and run `pkg update` again.

    3. **Step 3: Clear Cache and Rebuild:**
      apt clean && apt update
    4. **Step 4: Fix Expired Key Error (If applicable):** Run `pkg update && pkg upgrade`. If the error persists, check GitHub for the latest manual key refresh command.

Chapter 1: Execution, Commands, and File Permissions

When something doesn't run, check the three Ps: **Package**, **Path**, and **Permissions**.

  • Error: `bash: git: command not found` (or `wget`, `nano`, `python`, etc.)

    **Cause:** The package for the command is not installed, or its path is not in the system's PATH variable.

    **Solution:**

    1. **Install the package:**
      pkg install
    2. **If still not found**, verify the command is actually in the standard executable paths:
      echo $PATH
  • Error: `Permission denied` when accessing Android storage (SD Card, Downloads, etc.).

    **Cause:** Termux does not have Android's storage permission yet.

    **Solution: Setup Storage Access:**

    1. **Run the setup utility:**
      termux-setup-storage
    2. **Crucial Caveat:** Only run large tools or install complex packages within the **Termux Home Directory** (`~`). External storage paths often cause execution failures (e.g., Git failures).

Chapter 2: Networking and DNS Issues

When network requests fail, the problem is rarely the Termux application itself, but often DNS resolution.

  • Problem: Termux cannot resolve domain names or download packages (DNS Failure).

    **Cause:** Your device's DNS is not being picked up correctly, or the mirror is blocked.

    **Solution: Force Google DNS** (Most Reliable Fix):

    rm $PREFIX/etc/resolv.conf && echo "nameserver 8.8.8.8" > $PREFIX/etc/resolv.conf
  • Error: Git clone fails with `ssl certificate problem: unable to get local issuer certificate`.

    **Cause:** Git needs the local certificate store updated to trust HTTPS connections.

    **Solution: Install and Update CA Certificates:**

    pkg install ca-certificates

Chapter 3: Stability, Background, and API Issues

Addressing unexpected application closures and issues with interacting with the Android system.

  • Problem: Termux freezing, crashing, or processes being killed unexpectedly (Phantom Processes).

    **Cause:** Aggressive battery optimization settings on Android, especially Android 10+ (Phantom Processes).

    **Solution:**

    1. **Disable battery optimization** for Termux in Android settings.
    2. **Use `tmux` or `screen`** to manage sessions that survive disconnections:
      pkg install tmux && tmux new -s persistent-session
    3. **If totally frozen**, force-kill all Termux processes:
      pkill -f "termux"
  • Error: Termux:API commands not found or not working (e.g., `termux-battery-status`).

    **Cause:** Requires two separate components: the Termux package and the separate Android app.

    **Solution:**

    1. **Install the API package in Termux:**
      pkg install termux-api
    2. **Install the separate Termux:API app** from F-Droid or GitHub.

Chapter 4: Package-Specific and Advanced Build Errors

Common issues when compiling tools or installing Python dependencies.

  • Error: `pip install` fails with `Error: command 'gcc' failed` or dependencies missing.

    **Cause:** Python packages need to compile C/C++ dependencies (`build-essentials`).

    **Solution: Install development tools first:**

    pkg install python clang make libcrypt-dev

    These packages provide the compiler and necessary headers for many libraries to build correctly.

  • Error: `dpkg: error processing package (--install): : No such file or directory`.

    **Cause:** Corrupted package metadata, usually after a failed upgrade or installation.

    **Solution: Force a fix and clean up:**

    dpkg --configure -a && pkg install --fix-missing
  • Error: `Segmentation fault` or `Illegal Instruction` during program execution.

    **Cause:** The tool is not compiled for your device's CPU architecture (e.g., using an x86 tool on an ARM device).

    **Solution:** Use the command `uname -m` to verify your architecture and only install packages compatible with that architecture (aarch64, armv7l, or i686).

Chapter 5: Terminal Display and Configuration

Fixing visual errors, shell startup issues, and keyboard problems.

  • Problem: Changes to `.bashrc` or `.zshrc` are ignored until I restart Termux.

    **Cause:** Shell configuration files must be manually "sourced" to apply changes immediately.

    **Solution: Manually reload the configuration:**

    source ~/.bashrc (or source ~/.zshrc)
  • Problem: Unreadable characters or missing symbols when editing files (Text Encoding).

    **Cause:** The terminal environment is not configured for Unicode (UTF-8) correctly.

    **Solution: Set the correct environment variable in your shell config (`~/.bashrc`):**

    export LANG=en_US.UTF-8
  • Problem: I can't type special keys (like F-keys, Esc, or Ctrl+C) easily.

    **Cause:** Termux uses volume keys and swipes for special characters.

    **Solution: Use built-in key combinations and the extra keys view:**

    1. **`CTRL` Key:** Press the **Volume Down** button and then the character key (e.g., `Vol Down` + `C` for Ctrl+C).
    2. **`ESC` Key:** Press the **Volume Up** button and then the `E` key.

Chapter 6: Advanced Environment and Execution

Understanding the Termux sandbox, its unique directory structure, and advanced execution utilities.

Termux Key Environment Variables

Termux is a non-root user-space environment. All tools are installed in a specific location that differs from a standard Linux system.

  • **`$PREFIX`**: The Termux installation root (`/data/data/com.termux/files/usr`). All binaries and libraries are found here.
  • **`$HOME`**: The user's home directory (`/data/data/com.termux/files/home`). All personal files and user-installed tools should live here.
  • Error: `sudo: command not found` or need for root permissions.

    **Cause:** Termux is not a rooted environment by default and cannot use superuser commands.

    **Solution: Use Proot or Fakeroot:**

    1. **To emulate a full Linux root environment (chroot):**
      pkg install proot

      Run a rootless Linux distribution "chroot" using `proot`.

    2. **For building packages requiring root ownership:**
      pkg install fakeroot

      Use `fakeroot` to create archives with specific permissions.

  • Problem: Program starts and immediately fails silently (or `No such file or directory`).

    **Cause:** The program may be attempting a system call (like file access or network setup) that is blocked or missing a dependency, but it's not printing a useful error.

    **Solution: Trace System Calls with `strace`:**

    pkg install strace && strace -f

    `strace` will print a line-by-line log of all system interactions, often revealing the exact "Permission denied" or path error that caused the crash.

Chapter 7: Customization and Graphical Environment (X11)

Installing alternative shells and setting up a basic graphical desktop for X-Window applications.

  • Problem: I want better features and customization than the default Bash shell.

    **Cause:** Bash is the standard, but shells like Zsh offer advanced autocompletion and extensibility via plugins.

    **Solution: Install Zsh and Switch Shells:**

    1. **Install the package:**
      pkg install zsh
    2. **Switch your default shell:**
      chsh -s zsh

      You must reload Termux for this change to take effect.

    3. **Customize the prompt (e.g., in `~/.zshrc`):**
      export PROMPT='%n@%m:%~%# '
  • Problem: I need to run a graphical (X-Window) Linux application.

    **Cause:** Graphical applications require an X Server running on the Android device to render the output.

    **Solution: Setup X11 Environment:**

    1. **Install the X11 repository and client package:**
      pkg install x11-repo && pkg install termux-x11-nightly
    2. **Install the separate Termux:X11 Android app** from F-Droid or GitHub.
    3. **Set the display variable and run an app (e.g., `xeyes`):**
      export DISPLAY=':0' && pkg install xeyes && xeyes

      The Termux:X11 app must be running in the background for this to work.

Final Word on Termux

Termux is a powerful and low-level environment. When troubleshooting, always proceed step-by-step, starting with the correct installation source (F-Droid) and proper permissions. If an error persists, always check the **Termux Wiki** or **GitHub Issues**.

Κεφάλαιο 0: Κρίσιμα Θέματα Εγκατάστασης και Αποθετηρίων

Όλα τα θέματα σταθερότητας του Termux συνήθως οφείλονται σε λανθασμένη πηγή εγκατάστασης ή σε μια παρωχημένη λίστα πακέτων. Επιλύστε αυτά τα θεμελιώδη ζητήματα πρώτα.

ΚΡΙΣΙΜΗ ΠΡΟΕΙΔΟΠΟΙΗΣΗ: Η Έκδοση του Google Play Store είναι Ξεπερασμένη!

Η έκδοση του **Termux** από το Google Play Store είναι μόνιμα παρωχημένη και μη υποστηριζόμενη λόγω αλλαγών στην πολιτική του Android. Είναι **υποχρεωτικό** να χρησιμοποιήσετε την έκδοση από το **F-Droid** ή τις επίσημες κυκλοφορίες του **GitHub**.

  • **Λύση:** Απεγκατάσταση παλιάς έκδοσης, εγκατάσταση από **F-Droid**.
  • Πρόβλημα: Σφάλματα Ενημέρωσης Πακέτων (`404 Not Found`, `repository not signed`, `E: Unable to locate`).

    **Αιτία:** Ελαττωματικός σύνδεσμος αποθετηρίου, πρόβλημα με τον διακομιστή mirror, ή χρήση λάθος έκδοσης εφαρμογής.

    **Λύση:**

    1. **Βήμα 1: Ενημέρωση & Αναβάθμιση (Απαραίτητο):**
      pkg update && pkg upgrade -y
    2. **Βήμα 2: Αλλαγή Mirror Αποθετηρίου:**
      termux-change-repo

      Επιλέξτε ένα διαφορετικό γεωγραφικό mirror (π.χ. Grimler ή Alabugin) και εκτελέστε ξανά το `pkg update`.

    3. **Βήμα 3: Εκκαθάριση Cache και Αναδόμηση:**
      apt clean && apt update
    4. **Βήμα 4: Διόρθωση Σφάλματος Λήξης Κλειδιού (Εάν χρειάζεται):** Εκτελέστε `pkg update && pkg upgrade`. Εάν το σφάλμα επιμένει, ελέγξτε το GitHub για την τελευταία εντολή μη αυτόματης ανανέωσης κλειδιού.

Κεφάλαιο 1: Εκτέλεση, Εντολές και Δικαιώματα Αρχείων

Όταν κάτι δεν εκτελείται, ελέγξτε τα τρία Π: **Πακέτο**, **Διαδρομή** (Path) και **Πρόσβαση** (Permissions).

  • Σφάλμα: `bash: git: command not found` (ή `wget`, `nano`, `python`, κ.λπ.)

    **Αιτία:** Το πακέτο για την εντολή δεν είναι εγκατεστημένο ή η διαδρομή του δεν βρίσκεται στη μεταβλητή PATH του συστήματος.

    **Λύση:**

    1. **Εγκαταστήστε το πακέτο:**
      pkg install <όνομα-εντολής>
    2. **Εάν εξακολουθεί να μην βρίσκεται**, επαληθεύστε ότι η εντολή βρίσκεται πράγματι στις τυπικές διαδρομές εκτέλεσης:
      echo $PATH
  • Σφάλμα: `Permission denied` κατά την πρόσβαση στην αποθήκευση του Android (Κάρτα SD, Λήψεις, κ.λπ.).

    **Αιτία:** Το Termux δεν έχει ακόμη τα δικαιώματα πρόσβασης αποθήκευσης του Android.

    **Λύση: Ρύθμιση Πρόσβασης Αποθήκευσης:**

    1. **Εκτελέστε το βοηθητικό πρόγραμμα ρύθμισης:**
      termux-setup-storage
    2. **Κρίσιμη Παρατήρηση:** Εκτελέστε μεγάλα εργαλεία ή εγκαταστήστε σύνθετα πακέτα μόνο μέσα στον **Αρχικό Κατάλογο του Termux** (`~`). Οι διαδρομές εξωτερικής αποθήκευσης συχνά προκαλούν σφάλματα εκτέλεσης (π.χ. σφάλματα Git).

Κεφάλαιο 2: Θέματα Δικτύωσης και DNS

Όταν αποτυγχάνουν οι αιτήσεις δικτύου, το πρόβλημα σπάνια είναι η ίδια η εφαρμογή Termux, αλλά συχνά η επίλυση ονομάτων (DNS).

  • Πρόβλημα: Το Termux δεν μπορεί να επιλύσει ονόματα τομέα ή να κατεβάσει πακέτα (Αποτυχία DNS).

    **Αιτία:** Οι διακομιστές DNS της συσκευής σας δεν λαμβάνονται σωστά ή ο καθρέφτης είναι μπλοκαρισμένος.

    **Λύση: Επιβολή Google DNS** (Η πιο Αξιόπιστη Διόρθωση):

    rm $PREFIX/etc/resolv.conf && echo "nameserver 8.8.8.8" > $PREFIX/etc/resolv.conf
  • Σφάλμα: Το Git clone αποτυγχάνει με `ssl certificate problem: unable to get local issuer certificate`.

    **Αιτία:** Το Git χρειάζεται ενημέρωση του τοπικού καταστήματος πιστοποιητικών για να εμπιστεύεται τις συνδέσεις HTTPS.

    **Λύση: Εγκατάσταση και Ενημέρωση Πιστοποιητικών CA:**

    pkg install ca-certificates

Κεφάλαιο 3: Σταθερότητα, Ιστορικό και Θέματα API

Αντιμετώπιση απροσδόκητων κλεισιμάτων εφαρμογών και προβλημάτων αλληλεπίδρασης με το σύστημα Android.

  • Πρόβλημα: Το Termux παγώνει, κρασάρει ή οι διεργασίες τερματίζονται απροσδόκητα (Διεργασίες Φάντασμα).

    **Αιτία:** Επιθετικές ρυθμίσεις βελτιστοποίησης μπαταρίας στο Android, ειδικά στο Android 10+ (Phantom Processes).

    **Λύση:**

    1. **Απενεργοποιήστε τη βελτιστοποίηση μπαταρίας** για το Termux στις ρυθμίσεις του Android.
    2. **Χρησιμοποιήστε `tmux` ή `screen`** για τη διαχείριση περιόδων λειτουργίας που επιβιώνουν από αποσυνδέσεις:
      pkg install tmux && tmux new -s persistent-session
    3. **Εάν έχει παγώσει εντελώς**, τερματίστε βίαια όλες τις διεργασίες Termux:
      pkill -f "termux"
  • Σφάλμα: Οι εντολές Termux:API δεν βρίσκονται ή δεν λειτουργούν (π.χ., `termux-battery-status`).

    **Αιτία:** Απαιτούνται δύο ξεχωριστά στοιχεία: το πακέτο Termux και η ξεχωριστή εφαρμογή Android.

    **Λύση:**

    1. **Εγκαταστήστε το πακέτο API στο Termux:**
      pkg install termux-api
    2. **Εγκαταστήστε την ξεχωριστή εφαρμογή Termux:API** από το F-Droid ή το GitHub.

Κεφάλαιο 4: Ειδικά Πακέτα και Προχωρημένα Σφάλματα Δόμησης

Συνήθη ζητήματα κατά τη μεταγλώττιση εργαλείων ή την εγκατάσταση εξαρτήσεων Python.

  • Σφάλμα: Το `pip install` αποτυγχάνει με `Error: command 'gcc' failed` ή έλλειψη εξαρτήσεων.

    **Αιτία:** Τα πακέτα Python χρειάζεται να μεταγλωττίσουν εξαρτήσεις C/C++ (`build-essentials`).

    **Λύση: Εγκαταστήστε πρώτα τα εργαλεία ανάπτυξης:**

    pkg install python clang make libcrypt-dev

    Αυτά τα πακέτα παρέχουν τον μεταγλωττιστή και τις απαραίτητες κεφαλίδες για να δομηθούν σωστά πολλές βιβλιοθήκες.

  • Σφάλμα: `dpkg: error processing package <πακέτο> (--install): <διαδρομή>: No such file or directory`.

    **Αιτία:** Κατεστραμμένα μεταδεδομένα πακέτου, συνήθως μετά από μια αποτυχημένη αναβάθμιση ή εγκατάσταση.

    **Λύση: Επιβολή διόρθωσης και καθαρισμός:**

    dpkg --configure -a && pkg install --fix-missing
  • Σφάλμα: `Segmentation fault` ή `Illegal Instruction` κατά την εκτέλεση προγράμματος.

    **Αιτία:** Το εργαλείο δεν έχει μεταγλωττιστεί για την αρχιτεκτονική CPU της συσκευής σας (π.χ., χρήση ενός εργαλείου x86 σε συσκευή ARM).

    **Λύση:** Χρησιμοποιήστε την εντολή `uname -m` για να επαληθεύσετε την αρχιτεκτονική σας και μόνο εγκαταστήστε πακέτα συμβατά με αυτήν την αρχιτεκτονική (aarch64, armv7l ή i686).

Κεφάλαιο 5: Οθόνη Τερματικού και Διαμόρφωση

Διόρθωση οπτικών σφαλμάτων, προβλημάτων εκκίνησης κελύφους και ζητημάτων πληκτρολογίου.

  • Πρόβλημα: Οι αλλαγές σε `.bashrc` ή `.zshrc` αγνοούνται μέχρι να επανεκκινήσω το Termux.

    **Αιτία:** Τα αρχεία διαμόρφωσης κελύφους πρέπει να "πηγάζουν" (source) χειροκίνητα για να εφαρμοστούν οι αλλαγές αμέσως.

    **Λύση: Φορτώστε ξανά χειροκίνητα τη διαμόρφωση:**

    source ~/.bashrc (or source ~/.zshrc)
  • Πρόβλημα: Μη αναγνώσιμοι χαρακτήρες ή ελλιπή σύμβολα κατά την επεξεργασία αρχείων (Κωδικοποίηση Κειμένου).

    **Αιτία:** Το περιβάλλον τερματικού δεν είναι διαμορφωμένο σωστά για Unicode (UTF-8).

    **Λύση: Ορίστε τη σωστή μεταβλητή περιβάλλοντος στη διαμόρφωση κελύφους (`~/.bashrc`):**

    export LANG=en_US.UTF-8
  • Πρόβλημα: Δεν μπορώ να πληκτρολογήσω εύκολα ειδικά πλήκτρα (όπως F-keys, Esc, ή Ctrl+C).

    **Αιτία:** Το Termux χρησιμοποιεί πλήκτρα έντασης ήχου και χειρονομίες για ειδικούς χαρακτήρες.

    **Λύση: Χρησιμοποιήστε τους ενσωματωμένους συνδυασμούς πλήκτρων και την προβολή επιπλέον πλήκτρων:**

    1. **Πλήκτρο `CTRL`:** Πατήστε το πλήκτρο **Μείωσης Έντασης** (Volume Down) και μετά το πλήκτρο χαρακτήρα (π.χ., `Vol Down` + `C` για Ctrl+C).
    2. **Πλήκτρο `ESC`:** Πατήστε το πλήκτρο **Αύξησης Έντασης** (Volume Up) και μετά το πλήκτρο `E`.

Κεφάλαιο 6: Προηγμένο Περιβάλλον και Εκτέλεση

Κατανόηση του sandbox του Termux, της μοναδικής δομής καταλόγων και των προηγμένων βοηθητικών προγραμμάτων εκτέλεσης.

Βασικές Μεταβλητές Περιβάλλοντος Termux

Το Termux είναι ένα περιβάλλον χώρου χρήστη (user-space) χωρίς root από προεπιλογή. Όλα τα εργαλεία είναι εγκατεστημένα σε μια συγκεκριμένη θέση που διαφέρει από ένα τυπικό σύστημα Linux.

  • **`$PREFIX`**: Ο ριζικός κατάλογος εγκατάστασης Termux (`/data/data/com.termux/files/usr`). Όλα τα δυαδικά και βιβλιοθήκες βρίσκονται εδώ.
  • **`$HOME`**: Ο αρχικός κατάλογος του χρήστη (`/data/data/com.termux/files/home`). Όλα τα προσωπικά αρχεία και τα εργαλεία που εγκαθιστά ο χρήστης πρέπει να βρίσκονται εδώ.
  • Σφάλμα: `sudo: command not found` ή ανάγκη για δικαιώματα root.

    **Αιτία:** Το Termux δεν είναι περιβάλλον με root από προεπιλογή και δεν μπορεί να χρησιμοποιήσει εντολές υπερχρήστη.

    **Λύση: Χρήση Proot ή Fakeroot:**

    1. **Για εξομοίωση ενός πλήρους περιβάλλοντος root Linux (chroot):**
      pkg install proot

      Εκτελέστε μια διανομή Linux χωρίς root "chroot" χρησιμοποιώντας το `proot`.

    2. **Για τη δόμηση πακέτων που απαιτούν ιδιοκτησία root:**
      pkg install fakeroot

      Χρησιμοποιήστε το `fakeroot` για να δημιουργήσετε αρχεία με συγκεκριμένα δικαιώματα.

  • Πρόβλημα: Το πρόγραμμα ξεκινά και αποτυγχάνει αμέσως σιωπηλά (ή `No such file or directory`).

    **Αιτία:** Το πρόγραμμα ενδέχεται να επιχειρεί μια κλήση συστήματος (όπως πρόσβαση σε αρχείο ή ρύθμιση δικτύου) που είναι αποκλεισμένη ή λείπει μια εξάρτηση, αλλά δεν εκτυπώνει ένα χρήσιμο σφάλμα.

    **Λύση: Ανίχνευση Κλήσεων Συστήματος με `strace`:**

    pkg install strace && strace -f <εντολή-που-αποτυγχάνει>

    Το `strace` θα εκτυπώσει ένα λεπτομερές αρχείο καταγραφής όλων των αλληλεπιδράσεων του συστήματος, αποκαλύπτοντας συχνά το ακριβές σφάλμα "Permission denied" ή σφάλμα διαδρομής που προκάλεσε την κατάρρευση.

Κεφάλαιο 7: Προσαρμογή και Γραφικό Περιβάλλον (X11)

Εγκατάσταση εναλλακτικών κελυφών και ρύθμιση μιας βασικής γραφικής επιφάνειας εργασίας για εφαρμογές X-Window.

  • Πρόβλημα: Θέλω καλύτερες λειτουργίες και προσαρμογή από το προεπιλεγμένο κέλυφος Bash.

    **Αιτία:** Το Bash είναι το πρότυπο, αλλά τα κελύφη όπως το Zsh προσφέρουν προηγμένη αυτόματη συμπλήρωση και επεκτασιμότητα μέσω προσθηκών.

    **Λύση: Εγκαταστήστε το Zsh και Αλλάξτε Κελύφη:**

    1. **Εγκαταστήστε το πακέτο:**
      pkg install zsh
    2. **Αλλάξτε το προεπιλεγμένο κέλυφος:**
      chsh -s zsh

      Πρέπει να επανεκκινήσετε το Termux για να τεθεί σε ισχύ αυτή η αλλαγή.

    3. **Προσαρμόστε την προτροπή (π.χ. στο `~/.zshrc`):**
      export PROMPT='%n@%m:%~%# '
  • Πρόβλημα: Πρέπει να εκτελέσω μια γραφική εφαρμογή Linux (X-Window).

    **Αιτία:** Οι γραφικές εφαρμογές απαιτούν έναν X Server που εκτελείται στη συσκευή Android για να αποδώσει την έξοδο.

    **Λύση: Ρύθμιση Περιβάλλοντος X11:**

    1. **Εγκαταστήστε το αποθετήριο X11 και το πακέτο πελάτη:**
      pkg install x11-repo && pkg install termux-x11-nightly
    2. **Εγκαταστήστε την ξεχωριστή εφαρμογή Android Termux:X11** από το F-Droid ή το GitHub.
    3. **Ορίστε τη μεταβλητή εμφάνισης και εκτελέστε μια εφαρμογή (π.χ., `xeyes`):**
      export DISPLAY=':0' && pkg install xeyes && xeyes

      Η εφαρμογή Termux:X11 πρέπει να εκτελείται στο παρασκήνιο για να λειτουργήσει αυτό.

Το Termux είναι ένα ισχυρό περιβάλλον χαμηλού επιπέδου. Κατά την αντιμετώπιση προβλημάτων, προχωρήστε πάντα βήμα προς βήμα, ξεκινώντας από τη σωστή πηγή εγκατάστασης (F-Droid) και τα σωστά δικαιώματα. Εάν ένα σφάλμα επιμένει, ελέγξτε πάντα το **Termux Wiki** ή τα **GitHub Issues**.
